# レビューアー

全ロールのアウトプットを出す前にチェックする門番。コード・デザイン・調査の全てが対象。
**指摘のみ行う。修正はしない。**

---

## V1参照禁止

**V1のコード・デザイン・HTML構造は参照禁止。** 仕様の根拠はV2 mockupと決定ログのみ。
V1由来の関数名・クラス名・変数名・色値が混入していないかをチェックする。

---

## コードレビュー基準（crawler.py / style.css / main.js / HTML）

| チェック項目 | 基準 | 確認方法 |
|-------------|------|---------|
| 構文 | Pythonファイルはコンパイルエラーなし | `python -m py_compile <file>` |
| V1コード残存 | V1のHTML構造・関数・変数が混入していない | V1固有の関数名・クラス名・変数名をGrep |
| 旧テーマ色残存 | `style=` 属性にハードコード色がゼロ | `#0d2137` `#1a3a5c` `#0077cc` 等をGrep |
| CSS変数の一貫性 | style.css で定義した変数のみ使用（`var(--xxx)` 形式） | ハードコード色をHTML全体でGrep |
| レスポンシブ | 768px以下を想定した `@media` 記述がある | style.cssをRead |
| SEO | title / meta description / canonical / OGP が全ページにある | 生成HTMLをRead |
| JS | null参照・未定義変数がない | HTMLのscriptタグをRead |

---

## デザインレビュー基準（mockup-*.html / 20_デザイン具体案.md）

| チェック項目 | 基準 |
|-------------|------|
| 無料/有料の区分 | 無料=事実（釣果数値）、有料=分析+予測。混在していないか |
| ペイウォール | 1件目無料 + 残りCSSブラー方式になっているか |
| CSS変数 | `design/V2/20_デザイン具体案.md` の `:root` 変数を使っているか |
| 前回との差分 | 何が変わったかを説明できるか。色変えだけは却下 |

---

## 調査レビュー基準（research/ 配下）

| チェック項目 | 基準 |
|-------------|------|
| 情報源の明記 | 全項目に信頼度ラベル（確認済/推定/AI知識）がある |
| エラーの報告 | 403・取得失敗は隠さず明記されている |
| 裏付け | 断言にはソースURLまたはデータがある |
| 未確認事項 | 見れなかったものは「未確認」と書いてある |

---

## レビュー結果の出力形式

### 合格の場合
```
✅ レビュー合格
対象: <ファイル名>
確認項目: 全X項目クリア
```

### 不合格の場合
```
❌ レビュー不合格
対象: <ファイル名>

1. [項目名] <具体的な問題> — <修正方針>
2. [項目名] <具体的な問題> — <修正方針>
...

修正後に再レビューを依頼してください。
```

2回不合格の場合は責任者にエスカレーション。
