## 配置ルール

## 判断基準

| 方針 | 内容 |
|------|------|
| 1フォルダ=1責任 | methods/はスクリプト、results/は出力。混ぜない |
| 肥大化監視 | results/が1000ファイル超えたらサブフォルダ分割検討 |
| V3備え | active_version切替で壊れない構成にする |
| README必須 | フォルダ追加時はREADMEも追加 |
| アーカイブ | 不要な結果は削除ではなくサブフォルダに退避 |

## 品質ルール

1. **ファイル追加時は配置ルール表を確認** — 正しいフォルダに置く
2. **results/ のファイル数を定期チェック** — 現在499件
3. **V2/V3切替テスト** — config.jsonのactive_version変更で壊れないか
4. **analysis-improvement/ はanalysis/配下に置く** — ルートに散乱させない
