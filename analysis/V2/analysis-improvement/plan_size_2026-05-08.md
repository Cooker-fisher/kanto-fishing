## 背景・新設計の根拠

### 直前セッションでの確認事項

1. **中央予測モデル (pred_avg) の精度は良好**：size_avg wMAPE P50=7.3%（diag_size_2026-05-08.md）
2. **問題は「予測値→ レンジ展開」のロジック**：`pred_lo = pred_avg - MAE` で MAE 1.75cm に対して actual の min/max 幅は 7cm（avg の 25%）と乖離
3. **構造的バイアス**：pred_lo - actual_min = +5.25cm（実測 +4.90cm・90.3% で正）

### ユーザーのドメイン知識（決定的）

> 「型については MIN〜MAX で MAX がお化けみたいな大物をひっかけることがあり、それが狂わす要因。MAX のお化けは予測対象外で、当たらなくても誰も文句言わない。MIN（下限）と MAX の重要度は **非対称**」

→ cnt（匹数）の min/max は対称的に重要だが、size の min/max は **非対称**。

| 指標 | 重要度 | 理由 |
|---|---|---|
| **size_min（下限）** | 🔴 最重要 | 「思ったより小さい」が解約リスク最大 |
| size_max（上限）| 🟢 低い | お化け（偶発的大物）は当てなくても文句出ない |

## 新設計：実測幅ベース + size_max 外れ値除外

### コアアイデア

**中央予測モデル（既存・良好）はそのまま使い、min/max 幅だけを `combo_decadal` の旬別実測値から取得する。**

学習側（combo_deep_dive.py）には **新しいモデル追加不要**。集計列の追加のみ。

### 集計（combo_decadal 拡張）

| 列 | 計算方法 | 用途 |
|---|---|---|
| `avg_size_min` | 旬別 actual_size_min の中央値 | size_lo 計算用（厳格）|
| `avg_size_max` | 旬別 actual_size_max の **外れ値除外中央値**（P95 以上除く）| size_hi 計算用（控えめ・お化け除く）|

### 予測（predict_count.py）

```python
# 既存の中央予測（変更なし）
pred_avg = avg_size + 既存補正  # wMAPE 7.3% モデル

# 新規: 旬別実測幅でレンジ生成
size_lo = pred_avg - (avg_size - avg_size_min)      # 下限は厳格
size_hi = pred_avg + (avg_size_max - avg_size)      # 上限は典型値（お化け除く）

# 数学的等価形（実装ではこちらでもよい）:
# size_lo = pred_avg × (avg_size_min / avg_size)
# size_hi = pred_avg × (avg_size_max / avg_size)
```

### 評価軸（補遺3 整合）

| 指標 | 定義 | 重視度 |
|---|---|---|
| **promise_break** | actual_size_min < pred_lo（下限割れ）| 🔴 主指標 |
| over_expect | actual_size_max > pred_hi（お化けが出た）| 🟢 **記録するが KPI 外**（「お化け」は許容）|
| coverage | actual_size_avg ∈ [pred_lo, pred_hi] | 🟡 副指標 |

## 数値根拠（diag_size_2026-05-08.md より）

- 旬別 actual_size_min / size_avg の P50 = **0.75**
- 旬別 actual_size_max / size_avg の P50 = **1.25**

→ 旬別中央値で `size_lo = pred_avg × 0.75`、`size_hi = pred_avg × 1.25`

現状の `pred_avg ± mae`（avg=22cm の ±1.75cm = ±8%）から、**±25%** に幅が広がる。

### 期待効果

- promise_break **53% → 10〜15% 程度**を見込む（actual_min が pred_avg × 0.75 より下回るのは P10 以下）
- 中央予測の精度は変えないので wMAPE 7.3% は維持

## 実装上のリスクと対策

### リスク 1: 旬別データが少ないコンボで avg_size_min / max が不安定

- 旬別 n < 5 のセグメントでは中央値が外れ値の影響を受ける
- 対策: `MIN_N_DECADAL_SIZE = 5` を設けて、n < 5 の旬は全期間平均にフォールバック

### リスク 2: size_max 外れ値除外の閾値（P95）

- P95 以上は「お化け」と判定して除外
- ただし母数が少ない旬では P95 が機能しない
- 対策: n < 20 では「max 値そのまま」を使うか、または上位 5% カットを「最大値だけ除外」に置き換え
- 実装時に分布を見て決定（diag SQL で事前確認可）

### リスク 3: pred_avg がポイント別補正済みの場合の整合性

- predict_count.py は size_avg にポイント別補正（コミット 1ec169ab）を既に適用済み
- 補正後の pred_avg をベースに `± (avg_size_max - avg_size)` を加減
- 旬別実測幅は元の avg_size 比率で計算するため、ポイント補正で pred_avg が動いても比率は維持
- 設計上問題なし

### リスク 4: 複数旬にまたがる釣行日

- 1 つの target_date は 1 つの decade_no に属する
- combo_decadal の `avg_size_min[(fish, ship, decade_no)]` を引くだけで OK
