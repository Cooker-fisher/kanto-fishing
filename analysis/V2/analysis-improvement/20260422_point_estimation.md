# ポイント補完・推定手法まとめ（2026-04-22）

## 概要

point_rawが空のレコードに対して、3種類のクローラー・手法でポイントを補完・推定した。
結果として **幸栄丸のポイントカバレッジが 5% → 81% に拡大**し、ポイント別バックテストで 5/6コンボ +5pt以上の改善を達成。

---

## 1. 庄治郎丸：API直接クロール（shojiro_crawler.py）

### 手法
- shojiromaru.net の chowari.jp バックエンドAPI（jsonget.php）をPOST
- `ship_name` フィールドが**そのままポイント名**（例: 「平塚東沖」「平塚沖」）
- マッチング: `(date, tsuri_mono)` キーで catches_raw.json と突き合わせ

### 結果
- 補完件数: **1,634件**
- ポイント分布: 平塚沖・平塚東沖・平塚南沖 等
- ポイント別バックテスト: アマダイ×平塚沖 N=161 wMAPE=32.7%（BL2勝ち）等

---

## 2. 幸栄丸：3段階ポイント補完（koueimaru_crawler.py）

### 背景
- 幸栄丸のAPIは `ship_name` が空 → コメント（`choka_comment`）からのみポイント取得可能
- APIカバー範囲: select=1〜16（約15ヶ月）
- コメントにポイントキーワードがある割合: **19.4%のみ**（残80%は「内訳」や汎用コメント）

### 3段階補完ロジック

#### ① コメントキーワードマッチ
`choka_comment` をHTML除去した全文（`<br>`→スペース変換）に対してキーワード検索。
2行目以降も全文検索するため取りこぼしなし。

```python
POINT_KEYWORDS = {
    "北沖": "鹿島北沖",
    "真沖": "鹿島真沖",
    "南沖": "鹿島南沖",
    "魚礁": "鹿島魚礁",
    "大根": "大根",
    "湾内": "鹿島湾内",
}
```

#### ② 魚種デフォルトポイント（支配的ポイント ≥65%）
158件の実績データを分析し、魚種ごとの支配的ポイントを確認。

```python
FISH_DEFAULT_POINT = {
    "ヒラメ":   "鹿島北沖",  # 70% N=44
    "マハタ":   "鹿島北沖",  # 68% N=25
    "ヤリイカ": "鹿島真沖",  # 86% N=15
    "マダコ":   "鹿島真沖",  # 80% N=6
    "マダイ":   "鹿島南沖",  # 67% N=6
    "フグ":     "鹿島南沖",  # 57% N=7
    ...
}
```

**分析知見: ポイント決定の第一因子は魚種（ターゲット）**
- 同日に複数コースが出船（ヒラメ便→北沖、ワラサ便→真沖など）
- 気象は「どこに逃げるか」の補正要因

#### ③ 気象推定（分散コンボ: ワラサ/イナダ/ヒラマサ）
wave_height・SST・wind_dir から分岐:

| 条件 | 推定ポイント |
|------|-----------|
| wave ≥ 1.5m + 東寄り風（NE/E/N） | 鹿島北沖（東風・高波→北方向に避難） |
| wave ≥ 1.5m + 西/南西系風 | 鹿島真沖 |
| wave < 0.8m + SST ≥ 20℃ + 南寄り風 | 大根（夏季凪・青物回遊帯） |
| wave 0.8-1.5m + NW/W | 鹿島北沖 |
| wave < 0.8m + NE/E | 鹿島南沖 |
| その他 | 鹿島北沖（最頻） |

**気象データソース**: weather_cache.sqlite 近傍座標（lat=35.97, lon=140.77）の5-12時JST平均

### 補完結果
| 段階 | 件数 |
|------|------|
| ①コメントキーワード | 158件（既実行済み） |
| ②魚種デフォルト | 1,571件 |
| ③気象推定 | 323件 |
| **合計** | **2,052件 / 2,528件（81.2%）** |

## 3. 幸丸：水深帯を仮想ポイントとして使用（combo_deep_dive.py）

### 背景
- 幸丸（cb-sachimaru.com）は釣りビジョンに point_raw なし
- ただし **depth_min が93%カバー**（釣りビジョンのデータに水深記載あり）
- 幸栄丸は depth_min が0.4%のみ → 適用不可

### 手法
`load_records()` 内で `point_place1` が空かつ `depth_min` があるレコードに仮想ポイントを割当:

```python
if not point_place1:
    d_min = _float(row.get("depth_min"))
    if d_min:
        if d_min <= 40:    point_place1 = "浅場(~40m)"
        elif d_min <= 80:  point_place1 = "中深場(41-80m)"
        elif d_min <= 150: point_place1 = "深場(81-150m)"
        else:              point_place1 = "超深場(150m+)"
```

### 結果（H=0 cnt_avg、BL2比）

| コンボ | depth_band | N | wMAPE | BL2 | 改善 |
|--------|-----------|---|-------|-----|------|
| ヒラメ×幸丸 | 浅場(~40m) | 602 | 30.4% | 38.1% | **+7.7pt** |
| マダイ×幸丸 | 浅場(~40m) | 939 | 42.4% | 47.0% | **+4.6pt** |
| マハタ×幸丸 | 浅場(~40m) | 55 | 31.0% | 42.0% | **+11.0pt** |
| ヤリイカ×幸丸 | 深場(81-150m) | 57 | 51.3% | 62.5% | **+11.2pt** |

---

## 4. 幸栄丸の「ポイント×水深帯」組合せは不可

- 幸栄丸の depth_min カバレッジ: **0.4%** → 水深帯分割の軸がない
- 現状の「ポイント別バックテスト 5/6コンボ +5pt超」が実質的な上限

## ポイント補完手法の選択フロー

```
船宿のAPIにポイント情報がある？
  ├─ Yes: ship_name フィールドにある → 直接使用（庄治郎丸）
  │        コメントにキーワードがある → キーワード抽出（幸栄丸①）
  └─ No:
       depth_min のカバレッジが高い（>50%）？
         ├─ Yes → 水深帯を仮想ポイントとして使用（幸丸）
         └─ No:
              魚種ごとに支配的ポイントがある（≥65%）？
                ├─ Yes → 魚種デフォルト割当（幸栄丸②）
                └─ No → 気象条件（wave/SST/wind）で推定（幸栄丸③）
```
