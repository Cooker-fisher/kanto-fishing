## 2. 診断ファクト（コード読み確認済み）

### 2-1. MAX_FACTORS=12 設計と実績の整合性

`combo_deep_dive.py:326` `MAX_FACTORS = 12` は **2階建て因子設計の Tier2 上限** であり、`MAX_TIER2 = 12` と同義。

実装の実態:
- **Tier1 BASE_FACTORS (16因子)**: 潮汐・月齢・カレンダー・基本気象。常時確保・競合から保護
- **Tier2 残り因子 (最大 12因子)**: CMEMS含む。`|r|`降順で採用
- **MAX_CMEMS_DEFAULT=2**: CMEMS_FACTORS 集合内で上限 2（コンボ別 0/2/4/6 で自動最適化）

→ 実績 24因子 = 16 (BASE) + 8 (Tier2) で**設計通り**動作している。「上限機能してない」は誤読。

### 2-2. DO 採用漏れの真因

CMEMS_FACTORS 集合 (`combo_deep_dive.py:505-517`) に `do_surface, do_bottom` が含まれており、MAX_CMEMS_DEFAULT=2 で他CMEMS因子 (temp_100m_xxx・deepwater_score・no3_surface等) と競合している。

**DO 174コンボ採用時の他CMEMS採用上位:**
- `deepwater_score` 95件採用 (DO 関連だが別因子)
- `temp_100m_spring` 77件 / `temp_100m` 76件
- `no3_surface` 65件

→ DO がトップ2に入れず Tier2 から押し出されている可能性が高い。

### Phase 2: DO 因子を CMEMS_FACTORS から独立カテゴリ化（P2）

**設計変更:**

`combo_deep_dive.py` に以下を追加:

```python
# DO 専用カテゴリ（CMEMS から独立して 2 スロット確保）
# 理由: DO は底層酸素の直接指標で、青潮リスク・底魚生息可否を表す。
#       sla/chl/temp_100m_xxx 系（黒潮指標）と独立した情報源のため
#       同一カテゴリで競合させると押し出される（204コンボで採用漏れ実証済）。
DO_FACTORS = frozenset({"do_surface", "do_bottom"})
MAX_DO = 2  # do_surface + do_bottom 最大 2 因子（実質上限なし・両方使う）

# CMEMS_FACTORS から DO を除外
CMEMS_FACTORS = {
    "sla_avg", "chl_avg", "sss_avg", "kd490_surface",
    # "do_surface", "do_bottom",  ← DO_FACTORS に移管
    "temp_50m", "temp_100m", "temp_200m",
    "thermocline_depth", "no3_surface",
    "sla_delta", "chl_delta", "sla_monthly", "sla_lag30", "sla_approach_idx",
    "temp_100m_spring", "temp_100m_summer", "temp_100m_autumn", "temp_100m_winter",
    "temp_100m_bin", "chl_monthly",
    "kuroshio_score", "nutrient_score", "deepwater_score",
    "sss_delta_7d",
}
```

`_apply_factor_caps` に DO カウンタ追加:

```python
def _apply_factor_caps(factor_r_dict: dict, max_total: int = MAX_FACTORS,
                       max_cmems: int = MAX_CMEMS_DEFAULT,
                       max_do: int = MAX_DO,
                       corr_thr: float = 0.10) -> dict:
    base_selected = {f: r for f, r in factor_r_dict.items() if f in BASE_FACTORS}
    tier2_cands   = {f: r for f, r in factor_r_dict.items() if f not in BASE_FACTORS}

    tier2_sorted = sorted(tier2_cands.items(), key=lambda x: abs(x[1]), reverse=True)
    tier2_selected: dict = {}
    cmems_cnt = 0
    do_cnt = 0
    for fac, rv in tier2_sorted:
        if fac in DO_FACTORS:
            if do_cnt >= max_do:
                continue
            do_cnt += 1
        elif fac in CMEMS_FACTORS:
            if cmems_cnt >= max_cmems:
                continue
            cmems_cnt += 1
        tier2_selected[fac] = rv
        if len(tier2_selected) >= MAX_TIER2:
            break

    return {**base_selected, **tier2_selected}
```

**期待効果:**
- DO 採用コンボ: 174件 → 約 350件 (|r|>=0.30 で deep_params に存在する 378件のうちブラックリスト除外後)
- 採用される高相関 DO コンボの wMAPE 改善（avg|r|=0.45 ⇒ 直接寄与）
- 全体 wMAPE H=0 P50: 37.27% → 36.2〜36.5% (-0.8〜-1.0pt 見込み)

**リスク:**
- 過学習: DO はサンプル数が薄い船宿でノイズになる可能性。ただし `corr_thr = max(0.15, 1.96/√n)` の適応的閾値があるため小サンプル誤差は弾かれる
