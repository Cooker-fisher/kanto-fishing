## 1. メバル × 幸栄丸（最優先）

### 1-1. 基本情報

| 項目 | 値 |
|------|-----|
| n_records | 51 |
| エリア | 鹿島港（lat=36.05, lon=140.75） |
| time_slot | 全51件が空文字（time_slot未取得） |
| 釣期 | 2〜5月集中（2024-04: 14件, 2025-03: 15件） |
| use_fallback | 0 |

### 1-2. H別 wMAPE 推移（combo_backtest cnt_avg）

| H | wMAPE | BL2 | gap |
|---|-------|-----|-----|
| 0 | 36.5% | 42.4% | -5.9pt (勝ち) |
| 1 | 41.9% | 42.4% | -0.6pt (勝ち) |
| **3** | **54.2%** | 42.4% | **+11.7pt (負け)** |
| 7 | 52.6% | 42.4% | +10.2pt (負け) |
| 14 | 41.5% | 40.7% | +0.8pt (負け) |
| 21 | 54.6% | 42.9% | +11.7pt (負け) |
| 28 | 39.9% | 44.6% | -4.7pt (勝ち) |

**パターン: H3で急悪化（+17.6pt）。H7での悪化はH3と同水準（-1.6pt）。H14とH28は回復。**

これは「H>7での自動無効化が機能しているにもかかわらず悪い」ではなく、
**FAST因子がH=3〜7の範囲でノイズを注入している**パターン。

### 1-3. 採用FAST因子（combo_wx_params cnt_avg）

| rank | 因子 | r | 分類 |
|------|------|---|------|
| 1 | pressure_delta_48h | -0.388 | FAST |
| 2 | wind_speed_avg | +0.366 | FAST |
| 3 | precip_sum | +0.366 | FAST |
| 5 | precip_sum6 | -0.321 | FAST |
| 6 | wind_speed_max | +0.298 | FAST |
| 8 | precip_sum2 | -0.276 | FAST |

- **FAST因子数: 6 / 全15因子**（40%がFAST）
- FAST因子の |r| 平均: 0.336（SLOW最上位 moon_sin の 0.336 と同等）
- 月齢因子（moon_sin r=+0.336, moon_cos r=+0.178）が埋没している

### 1-4. 原因分析

メバル×幸栄丸（鹿島港・春季）のFAST因子問題は2層構造:

**層1: FAST因子が多く（6/15）、月齢主因子を数で埋没させている**
- 月齢は春メバルの接岸サイクルと連動するSLOW因子
- wind_speed_avg/precip_sum等は春季の天候変動を捉えるが、3日前の値は当日と無関係

**層2: H=3で悪化かつH=14で回復する非線形パターン**
- H=14, H=28 は FAST_MAX_H=7 によりFAST因子が除外済み → BL2圏内
- H=3 は FAST因子が有効だが3日前の気象は当日気象と相関が低い（特に降水ラグは6日後がピーク）
- 第三幸栄丸の per-combo FAST_MAX_H=3 は「月齢シグナルのみ使用」を意図しているが、
  幸栄丸（別船）は独立した問題として同様の対処が必要

**層3: time_slot 未取得（全51件が空文字）**
- メバルは夜行性/早朝便で釣果差が大きい
- time_slot が分離できていないため、time_slot交互作用は現時点で追加不可（将来課題）

### 1-5. per-combo FAST_MAX_H 推奨値

**推奨: FAST_MAX_H = 1**（H>1 でFAST因子全無効化）

根拠:
- H=1 でも -0.6pt の辛勝 → H=1以下のFAST情報は有効
- H=3 で +11.7pt の急悪化 → H=3のFAST情報は有害
- H=14 で +0.8pt（ほぼBL2同等）→ FASToなしなら月齢で追加勝ちが期待できる
- 第三幸栄丸の FAST_MAX_H=3 より更に短く設定する理由: 釣期が春季集中（台風なし）かつ
  メバルは光周性（月齢依存）が強く、気象変動への反応が遅い魚種

期待改善幅: H=7 で BL2同等〜-3pt程度（根拠: H=14 がBL2圏内のため、FAST除外により近似すると推定）

## 2. クロダイ × 平作丸（次優先）

### 2-1. 基本情報

| 項目 | 値 |
|------|-----|
| n_records | 53 |
| エリア | 久里浜港（lat未確認・神奈川） |
| 釣期 | 4〜5月のみ（春季限定） |

### 2-2. H別 wMAPE 推移

| H | wMAPE | BL2 | gap |
|---|-------|-----|-----|
| 0 | 40.3% | 47.9% | -7.6pt (勝ち) |
| 1 | 42.7% | 47.8% | -5.1pt (勝ち) |
| 3 | 40.0% | 48.1% | -8.1pt (勝ち) |
| **7** | **48.4%** | 48.2% | **+0.2pt (負け)** |
| 14 | 44.7% | 45.5% | -0.8pt (勝ち) |
| 21 | 54.6% | 43.9% | +10.7pt (負け) |
| 28 | 50.2% | 44.2% | +6.0pt (負け) |

**パターン: H3→H7の急悪化（+8.4pt）。H3 は -8.1pt の大勝ち。H7 だけ境界値で負け。**

H=21/28 の悪化は SLOW因子の問題（別議題）。

### 2-3. 採用FAST因子

| rank | 因子 | r | 注記 |
|------|------|---|------|
| 1 | precip_sum1 | +0.454 | 前日降水量 → H=3以上では当日と無関係 |
| 3 | prev_week_cnt | +0.335 | 前週釣果自己相関 → H>7で無効化されるはずだが実質H=4以降で陳腐化 |
| 5 | wind_dir_n | +0.323 | 風向 → 春の南風/北風パターンと連動するが3日先は不確定 |
| 7 | wind_dir_mode | -0.297 | 同上 |

**FAST因子数: 4 / 全15因子（wind系+prev_week_cnt+降水）**

### 2-4. 原因分析

H=7での境界負けの主因は `prev_week_cnt`:
- H=7 では「7日前の釣果」= 先々週の釣果を参照するため、現在の釣れ具合を反映できない
- FAST_MAX_H=7 の設計通り H>7 では無効化されるが、H=7 は有効な設定
- 春季4〜5月限定で n=53 と小さく、prev_week_cnt が春の「釣れる週が続く」現象を過学習

`precip_sum1` の問題:
- 前日降水 → H=3以上では「3日前の前日降水量」= 4日前の降水量を予測に使用
- これは当日気象と相関が薄く、ノイズになる可能性が高い

### 2-5. per-combo FAST_MAX_H 推奨値

**推奨: FAST_MAX_H = 3**

根拠:
- H=3 は -8.1pt の大勝ち → H=3以下ではFAST因子が有効
- H=7 では +0.2pt の境界負け → H=5〜6のどこかで悪化開始
- FAST_MAX_H=3 にすると H=7 の prev_week_cnt/wind_dir系が無効化され、
  SLOW因子（is_holiday/-0.426, tide_delta/-0.311）のみで予測 → BL2勝ちに戻る見込み

期待改善幅: H=7 で +0.2pt → -3〜-5pt（is_holidayとtide_deltaがSLOW上位因子のため）

## 3. イシダイ × 平作丸（保留）

### 3-1. 基本情報

| 項目 | 値 |
|------|-----|
| n_records | 76 |
| 釣期 | 11〜2月（冬季のみ） |

### 3-2. H別 wMAPE 推移

| H | wMAPE | BL2 | gap |
|---|-------|-----|-----|
| 0 | 47.3% | 49.4% | -2.1pt |
| 3 | 50.2% | 49.5% | +0.7pt |
| 7 | 50.8% | 49.2% | +1.6pt |
| 14 | 50.5% | 48.7% | +1.8pt |
| 21 | 57.4% | 49.7% | +7.6pt |

**パターン: 緩やか悪化。H0→H7 合計+3.5pt。H=3から既に負けている。**

### 3-3. 採用FAST因子と擬似相関の特定

| 因子 | r | 擬似相関の根拠 |
|------|---|-------------|
| typhoon_dist | +0.624 | **冬季専用コンボ。台風は7〜10月集中 → 釣期外=台風あり=距離大=釣れない時期という擬似相関** |
| typhoon_wind | +0.313 | 同上 |
| precip_sum | +0.287 | 冬季降水は雪/みぞれ → 実釣影響あり |
| wave_period_min | -0.243 | 波周期短い=北西風波=冬型気圧配置 → これは実質有効な季節シグナル |

`typhoon_dist`（r=+0.624）は全因子中最強だが**季節性擬似相関**:
- イシダイ釣期（11〜2月）= 台風シーズン外 → typhoon_dist が最大値（遠距離）
- 夏季（台風接近期）= イシダイ釣期外 → データに「釣れない日＋台風接近」が少ない
- つまり「台風が遠い日ほど釣れる」ではなく「台風が遠い＝冬季＝釣期内」の相関

### 3-4. per-combo FAST_MAX_H 推奨値と対処

**推奨: FAST_MAX_H の変更よりも FACTOR_BLACKLIST 追加が本質的対処**

根拠:
- typhoon_dist/typhoon_wind を除外すれば、FAST因子は precip_sum + wave_period_min の2個のみ
- この2因子は H=7でも一定の予報精度があり（wave_period は週間予報に含まれる）
- FAST_MAX_H を下げても typhoon_dist は H≤7 では引き続き使われてしまう

**提案実装:**
```python
FACTOR_BLACKLIST: dict[tuple, set] = {
    ...（既存）...
    ("イシダイ", "平作丸"): {"typhoon_dist", "typhoon_wind"},  # 冬季専用コンボ × 台風擬似相関
}
```

期待改善幅: H=7 で +1.6pt → -1〜-2pt（typhoon除外後の次位因子 water_color_pred_n が有効）

## 5. 実装優先度と実装コスト

### 優先実装（即効性あり）

**1. メバル × 幸栄丸: `_FAST_MAX_H_OVERRIDE[("メバル", "幸栄丸")] = 1`**
- 実装コスト: 1行追加（`combo_deep_dive.py` の `_FAST_MAX_H_OVERRIDE` dict）
- 再実行: メバル単体のみ（約30秒）
- 期待: H=7 で BL2圏内へ回復（+10.2pt → 0±3pt 程度）

**2. クロダイ × 平作丸: `_FAST_MAX_H_OVERRIDE[("クロダイ", "平作丸")] = 3`**
- 実装コスト: 1行追加
- 再実行: クロダイ単体のみ
- 期待: H=7 で -3〜-5pt 程度の改善

### 保留（FACTOR_BLACKLIST追加として別実装）

**3. イシダイ × 平作丸 / ヒラメ × 弘漁丸: typhoon FACTOR_BLACKLIST追加**
- gap が +1.6pt, +0.3pt と小さく、FAST_MAX_H変更よりFACTOR_BLACKLISTが適切
- FACTOR_BLACKLIST への typhoon_dist/typhoon_wind 追加は複数コンボに横断的に効く
- 実装コスト: FACTOR_BLACKLIST に2エントリ追加
- 再実行: イシダイ + ヒラメ の2種
- 期待改善幅: 小さい（gap自体が1.6pt以下）
