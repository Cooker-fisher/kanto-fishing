## MAPE改善の方向性（初期仮説）

1. **外れ値処理** — 極端な釣果データ（0匹・100匹超等）のクリッピングまたは除外
2. **変数選択の最適化** — 魚種×船宿ごとに有効な変数セットが異なる可能性
3. **ベースラインの精緻化** — combo_decadal（旬別平均）を月別・半旬別に細分化
4. **非線形効果の考慮** — 潮汐・月齢の閾値効果（大潮のみ有意等）
5. **欠航除外の徹底** — 欠航日データが平均を引き下げていないか

---

## 品質ルール

1. **改善効果はバックテスト（backtest_oos.py）で定量評価** — 「良くなった気がする」禁止
2. **変数の有効ホライズンを超えた使用は禁止** — D2でwave_heightを使わない
3. **過学習チェック** — インサンプルMAPE vs OOS MAPE の差が10%以上なら要見直し
4. **信頼区間を出す** — 点推定だけで判断しない

## ⚠️ バックテスト vs 予測の設計原則（2026/04/09 確定）

このプロジェクトには**2つの異なる精度検証フェーズ**がある。

### フェーズ1: バックテスト最適化（現行）
- **入力**: 気象**実績値**（actual wave height, actual SST 等）
- **目的**: 変数選択・相関検証・モデル構造の最適化
- **注意**: 実績値で最適化したパラメータが予測時にも最適とは限らない

### フェーズ2: 予測精度テスト（今後実装）
- **入力**: 気象**予報値**（Open-Meteo forecast API 等、誤差を含む）
- **目的**: 実運用での予測精度を検証
- **注意**: 予報誤差が大きい変数（波高・風速）ほど実績値との乖離が大きい

### 設計上の原則
| 変数種別 | 実績値との差 | バックテストでの扱い |
|---------|------------|------------------|
| is_holiday / 潮汐 / 月齢 | 誤差ゼロ（確定値） | そのまま使ってよい |
| SST / 気温 / 気圧 | 誤差小（週単位で安定） | バックテストで有効 |
| 波高 / 風速 | 誤差大（数日で急変） | H>7では無効化済み。予報誤差も要考慮 |

**コンボ別パラメータ最適化（wave_clamp等）は、予報値ベースの精度テストで検証してから本採用すること。**
