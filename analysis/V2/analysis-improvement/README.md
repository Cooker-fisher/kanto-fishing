## 目的

analysis/V2 の分析精度を改善し、予測モデル（D層）実装 → マネタイズ（月額500円/スポット100円）へつなげる。

### 4テーマ

1. **分析結果の改善** — MAPE 27〜55%の精度向上、combo_deep_dive全51魚種実行
2. **分析方法の改善** — スクリプト・手法の見直し、変数選択の最適化
3. **分析結果を下層にどう伝えるか** — C層→E層のデータフロー設計（forecast/、fish/*.html等への反映方法）
4. **予想結果の答え合わせ分析方法の検討** — 予測精度の検証フレームワーク、バックテスト手法の改善

## チーム構成（6エージェント + pm横断）

| エージェント | 役割 | 起動タイミング |
|------------|------|--------------|
| `analyst` | 分析スクリプト実装・バックテスト実行・統計設計（04+05統合） | 分析・実装フェーズ |
| `engineer` | B層CSV品質・GitHub CI・フォルダ構成（01+03+06統合） | CSV変更・コミット時 |
| `domain` | 釣りドメイン知識・E層表示設計（07+02統合） | 変数選択・表示設計時 |
| `code-reviewer` | Python構文・SQLロジック・条件分岐レビュー | スクリプト完成後（並列） |
| `stat-reviewer` | 統計的妥当性・過学習・有効ホライズンレビュー | スクリプト完成後（並列） |
| `data-reviewer` | join粒度・欠損値・集計単位レビュー | スクリプト完成後（並列） |
| `pm` | 横断管理（design/V2 + analysis/V2）・ワークフロー制御 | 常時 |

## ワークフロー

```
分析改善:
  pm → テーマ選択・エージェント起動
  analyst → 問題洗い出し・スクリプト実装（worktree）
  domain → 変数選択助言・表示要件定義
  engineer → CSV品質確認・コミット

レビュー（3エージェント並列）:
  code-reviewer + stat-reviewer + data-reviewer を同時起動
  全員合格 → engineer がコミット
  不合格あり → analyst が worktree で修正 → 再レビュー
```
