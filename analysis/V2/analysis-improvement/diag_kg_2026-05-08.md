## 4. kg_max ロングテール分析（「お化け」検出）

### 4.1 kg_max/kg_avg ヒストグラム（決定的）

| 範囲 | 件数 | 割合 |
|---|---|---|
| 0.0 - 1.5 | 14,133 | **52.83%** |
| 1.5 - 2.0 | 12,617 | **47.17%** |
| 2.0 - 2.5 | 0 | **0.00%** |
| 2.5 - 3.0 | 0 | 0.00% |
| 3.0+ | 0 | 0.00% |

### 4.2 重大な発見: kg は構造的に「お化け」が存在しない

**kg_max ≤ 2 × kg_avg = 2 × (kg_min + kg_max)/2 → kg_min ≥ 0**

これはデータ生成過程上の必然（kg_avg を `(kg_min + kg_max)/2` で計算しているため）。つまり:

- size: お化け = 単発的な極大値（P95 以上）が存在し、別ロジックで除外する設計が有効
- **kg: お化けは数学的に存在しない**（max は avg の最大 2 倍まで）

→ **size の P95 除外設計は kg には適用不要**。kg_max の P80 比率はそのまま使える。

### 4.3 P95 除外時の検証

P95 閾値 = 1.829。除外後 P80 = 1.667（除外前 1.700 と +0.033 のみの差）。**除外する意味は薄い**。

## 6. 期待効果の数値根拠

### 6.1 理論値（pred_avg が完全に正しい場合）

P20 比率を採用すると:
- promise_break = actual_kg_min < pred_kg_lo の確率
- pred_kg_lo = pred_avg × 0.30
- 実測 kg_min は分布全体から見ると下位 20% 区間付近
- → promise_break ≈ 20%（理論値）

### 6.2 中央予測精度の補正

kg_avg wMAPE P50 = 33.15% を考慮すると、pred_avg と actual_avg の系統的乖離が:
- pred_avg が 1.33 倍に膨らんでいるコンボ:
  - pred_kg_lo = (1.33 × actual_avg) × 0.3 = 0.4 × actual_avg
  - actual_kg_min ≈ 0.3 × actual_avg → 0.3 < 0.4 → promise_break **発生**
- pred_avg が 0.67 倍に縮んでいるコンボ:
  - pred_kg_lo = (0.67 × actual_avg) × 0.3 = 0.2 × actual_avg
  - actual_kg_min ≈ 0.3 × actual_avg → 0.3 > 0.2 → promise_break **不発**

→ wMAPE 33% の上振れ・下振れが対称分布なら、理論値 20% に **wMAPE の半分（≈16pt）が加算される**。

### 6.3 実測予想

| 指標 | 現状 | 期待値 |
|---|---|---|
| kg promise_break P50 | 55.56% | **35〜45%** |
| 改善幅 | - | **10〜20pt** |
| coverage | 0.679 | 0.40〜0.50（size 同様低下）|
| over_expect | 0.20 | 0.30〜0.40（size 同様増加）|
| winkler | 2.5 | 4〜6（区間幅広がる）|

size の +67pt 改善と比べて改善幅は小さいが、それでも実用的な改善範囲。

### 6.4 size との比較

| 観点 | size（実測）| kg（予想）|
|---|---|---|
| 旧 P50（同一定義）| 98.4% | 55.56%（既に Phase B-α' でなく旧 ±mae 設計）|
| 新 P50 | 31.25% | 35〜45% |
| 改善幅 | +67.2pt | +10〜20pt |
| 中央予測 wMAPE | 7.3% | 33.15% |
| 「お化け」非対称性 | あり（P95 除外要）| なし |
| 旬別データ充足 | 81.4% | 32.5%（MIN_N=10）/ 60.5%（MIN_N=5）|

## 7. 撤回基準の数値根拠（重要）

### 7.1 size の撤回基準（補遺7）

- promise_break 改善 ≥ 30pt（53% → 23% を超える改善が必要）
- 1コンボ動作確認エラー = 撤回

### 7.2 kg の撤回基準提案

中央予測精度低 (wMAPE 33%) を考慮すると、kg では size と同じ 30pt 基準は **過度に厳しい**。一方で「ほぼ変わらない」を許容すると改善検証として弱い。

| 基準値 | 期待値到達 | 妥当性 |
|---|---|---|
| 30pt（size 同等）| 55.56% → 25.56% 必要 | **過度に厳しい**（wMAPE 33% で物理的に困難）|
| 20pt | 55.56% → 35.56% 必要 | やや厳しいが達成可能 |
| **15pt** | 55.56% → **40.56% 必要** | **推奨**（10-20pt 期待中央付近）|
| 10pt | 55.56% → 45.56% | 緩すぎ（誤差レベル）|

**推奨: kg promise_break 改善 ≥ 15pt（55.56% → 40.56% 以下を到達基準とする）**

加えて以下を満たすこと:
- 1コンボ動作確認でエラーなし
- coverage が 0.30 以下に悪化しない（size 0.40 より緩い水準）

### 7.3 size との対比の整合

撤回基準 15pt は「中央予測 wMAPE 比例で size 30pt の半分」という根拠。size wMAPE 7.3% × 30pt vs kg wMAPE 33% × 7pt 程度が "wMAPE 中立な改善幅" だが、kg は実測幅が広い（P20=0.30 vs size 0.61）ためレンジ広げる効果は size より大きい → ネット 15pt 程度を狙える。

## 8. ドメイン観点の確認事項（pm/user 判断必要）

### 8.1 kg の非対称性

**データから確認済み**: kg は構造的にお化けが存在しない（kg_max ≤ 2×avg）。size の `size_max ≈ お化け・許容` のような非対称設計は kg では不要。

→ promise_break / over_expect は **対称評価**（両方とも主指標扱い）が妥当。size の補遺6 のように over_expect を許容ドメインとする必要なし。

### 8.2 kg 表示の優先度

ユーザー判断必要:
- E層 forecast/index.html や fish ページで kg は size と同等の優先度で表示すべきか
- 補遺3 で「重ねレンジ禁止」確定済み → 数 cnt / 型 size / 重 kg の 3レンジ並列表示
- マダコ・タチウオなど kg 表示しない魚種をどう扱うか（NULL のまま vs 非表示）

### 8.3 NULL 率 71% の解釈

ユーザー確認必要:
- NULL 率 71% は **kg 記載なしの釣果** が大半（小型魚は重量を書かない）
- 釣果報告に kg が含まれるか否かは魚種・船宿で偏りがある
- マダコ・ヒラメ・マダイは記載率高、アジ・サバ系は記載率低
- → **kg 表示は「kg を記録しているコンボのみ」**が正しい運用方針か確認

### 8.4 キハダマグロ等の例外コンボ

- pb=61.6% / coverage=0.251 / winkler=21.0 と他の魚種から大幅に外れる
- 大型魚で個体差が極端（10kg〜40kg）→ 中央予測の精度が物理的に低い
- Phase B-α' 適用後も改善幅は限定的
- 個別コンボ単位で「kg 表示しない」判定（use_fallback 系）の検討が必要か

## 9. リスクと対策

### 9.1 Phase B-α 撤回パターン非該当検証

補遺6 で確定の「4 回連続失敗パターン」（cnt_min 独立 → 05/07 ガード → 05/08 ガード → 05/08 Phase B-α）は **独立モデル化** に関するもの。本提案（実測幅ベース）は学習を変えず集計のみ追加するため、パターン非該当。

ただし実測幅ベースでも:
- size Phase B-α' は plan 期待 23% に対し実測 31%（コンボ単体 vs 全コンボ集計の数学的乖離）
- → kg でも同様の乖離が起こる可能性（理論 20% → 実測 35-45% の見積もりは、これを織り込み済み）

### 9.2 中央予測精度低の許容妥当性

kg wMAPE 33% は size 7.3% の 4.5倍。改善幅が小さくなるのは設計上必然で、撤回基準を緩めて受け入れる方針を Plan で明記する。

### 9.3 小サンプル統計の信頼性

- 旬別 n>=10 が 32.5% のみ → グローバル比率フォールバックが多用される
- グローバル P20=0.300 / P80=1.700 は CV が低い（P80 は CV=2%）→ 信頼性高
- ただし魚種別では分布が異なる（マダイ 1.7 vs キハダマグロ 1.36）
- → **魚種別 P20/P80 を別途持つ**設計の検討要（size には不要だったが kg には有用）

## 10. Plan 策定への申し送り

4. promise_break / over_expect は対称評価

1. **kg 表示の優先度**: 数 cnt / 型 size / 重 kg を並列表示で OK か
2. **NULL 71% コンボの扱い**: kg 表示しないコンボの判定基準
3. **キハダマグロ等の例外コンボ**: pb=61.6% / cov=0.25 / wink=21.0 を個別 use_fallback するか

### 10.4 Plan で含めない（別軸）
