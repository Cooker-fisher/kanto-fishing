## チェックリスト

### サンプル数

- [ ] n < 30 で回帰・相関分析をしていないか
- [ ] コンボ数（船宿×魚種）が少ない場合、結果を過信していないか
- [ ] 季節別分析で分割後のサンプルが極端に少なくないか

### 過学習

- [ ] インサンプルのみで評価していないか
- [ ] OOS（Out-of-Sample）検証結果が報告されているか
- [ ] train/test の分割が情報汚染なく行われているか
- [ ] 採用因子数が多すぎないか（MAX_FACTORS=10が守られているか）
- [ ] alpha_scale が 1.2 以下か

### 交差検証設計

- [ ] **leave-one-month-out CV** が実装されているか（2026/04/13確定）
- [ ] `train = date[:7] != test_month` が正しく実装されているか
- [ ] テスト月のデータが学習に混入していないか
- [ ] 最終パラメータ（wx_params）は全件で学習しているか（TRAIN_END廃止）

### 有効ホライズン違反

| 変数分類 | 有効H | 違反チェック |
|---------|-------|------------|
| SLOW（SST・気温・気圧・潮汐・月齢） | H≤28 | 28超で使っていないか |
| FAST（波・風・うねり・降水・潮流） | H≤7 | **H>7で使っていないか（特に重要）** |
| CALENDAR（土日・連休・乗っ込み） | 全H有効 | 問題なし |

**FAST変数をH>7で使うことは情報汚染と同等。即ブロック。**

### 評価指標

- [ ] wMAPEの重みが正しいか（cnt_avg基準か）
- [ ] 外れ値（cnt=0のボウズ日）が評価を歪めていないか
- [ ] promise_break_rate が計算されているか（PRIMARY KPI）
- [ ] BL-2勝率が報告されているか

### 情報汚染

- [ ] テスト月のアウトカム（釣果）が学習データに含まれていないか
- [ ] 未来の気象データを過去の予測に使っていないか
