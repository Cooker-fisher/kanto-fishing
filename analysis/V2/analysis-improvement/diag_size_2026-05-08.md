# size（型）精度問題 診断レポート（2026/05/08）

## サマリ（200 字以内）

size promise_break 53% の真因は **wMAPE ベース（size_avg モデル）の構造的バイアス**。
現行モデルは pred_avg ≈ actual_avg（wMAPE = 7.3% と優秀）だが、
pred_lo = size_avg_pred - MAE（中央 1.75cm）が actual_min（= actual_avg - actual_range/2 ≈ actual_avg - 7.0cm）より常に高くなる（bias +4.9cm）。
**Phase B で size_min / size_max を独立モデル化すれば構造的に解決可能。Go 判定。**

---

## 1. size_avg wMAPE の実数値分布

### size_avg（H=0, n>=30, n=193 コンボ）

| 指標 | P25 | P50 | P75 | P95 |
|------|-----|-----|-----|-----|
| size_avg wMAPE | 5.1% | 7.3% | 11.2% | 22.4% |
| **cnt_avg wMAPE（参考）** | 30.9% | 37.4% | 45.4% | 61.5% |

size の wMAPE は cnt の約 1/5 の水準。一見「精度良好」に見えるが、これが誤解の根源。

### 「wMAPE 良いのに promise_break 悪い」コンボ（H=0）

wMAPE < 30% かつ promise_break > 30% のコンボが **10 件**確認された（抜粋）：

| コンボ | wMAPE | promise_break | coverage |
|--------|-------|---------------|----------|
| タチウオ×○○○ | 1.2% | 67.9% | 100.0% |
| シロアマダイ×○○○ | 5.8% | 69.2% | 100.0% |
| マルイカ×梅花丸 | 17.5% | 65.2% | 95.7% |
| シロギス×○○ | 3.4% | 65.4% | 100.0% |
| ビシアジ×○○ | 12.0% | 66.1% | 80.5% |

wMAPE が小さい（center 予測精度良好）なのに promise_break が高い（下限が外れる）という「矛盾」が多数存在することを確認。これは構造的問題の典型症状。

## 2. 構造的問題の検証

### size_min / size_avg と size_max / size_avg の比率分布

| 指標 | P10 | P25 | P50 | P75 | P90 |
|------|-----|-----|-----|-----|-----|
| size_min / size_avg | 0.634 | 0.690 | **0.750** | 0.802 | 0.870 |
| size_max / size_avg | 1.130 | 1.198 | **1.250** | 1.310 | 1.366 |
| (size_max - size_min) / size_avg | 0.261 | 0.396 | **0.500** | 0.621 | 0.732 |

実際の size_min は avg の **75%** がP50値。つまり actual_range / 2 の中央値は **avg × 0.25 = 7.0cm**。

### cnt との比率比較

| | cnt_min/avg | cnt_max/avg |
|---|---|---|
| P50 | **0.280** | **1.782** |

size の P50 は 0.750 / 1.250 で **幅が cnt の 1/4 以下に狭い**。
cnt は ゼロからの大幅変動があるが、size は物理的制約（魚体サイズ）で変動幅が限定される。

## 3. size_min と size_max の相関

| 項目 | 値 |
|------|---|
| サンプル数 | 52,047 件 |
| Pearson r | **0.9416** |
| size_min == size_max（幅=0） | 3.3% |
| 幅 6-15cm | 55.2% |
| 幅 > 15cm | 38.3% |

**r = 0.94 は高い**。

独立モデル化への影響：
- 多重共線性リスクは存在するが、cnt_min / cnt_max の学習でも同様の高相関（報告値 r ≈ 0.8-0.9 程度）は許容されている
- size の場合「小さい魚は最大値も小さい」という自然法則的な相関であり、OLS 係数は識別可能
- 独立モデルで予測した size_lo / size_hi は同一の説明変数を使いながら係数が異なるだけで、理論上は収束可能
- **真の効果**: size_lo モデルは actual_size_min を直接ターゲットにするため、現行の「actual_avg - MAE」より低い値を予測できる

## 4. 予測値と actual_avg の系統的ずれ

### prediction_log（n=195 件）による実測

| 指標 | 値 |
|------|---|
| pred_avg - actual_avg 平均 | **-0.03 cm**（ほぼゼロ） |
| pred_avg - actual_avg 中央値 | **-0.20 cm** |
| pred_avg > actual_avg（過大） | 42.1% |

**center バイアスは存在しない**。wMAPE が小さい理由はここ。

| 指標 | 値 |
|------|---|
| pred_lo - actual_min 平均 | **+5.76 cm** |
| pred_lo - actual_min 中央値 | **+4.90 cm** |
| pred_lo > actual_min（下限過大） | **90.3%** |

**lo バイアスは+4.9cm（中央値）で顕著かつ一方向**。仮説を完全支持。

### 構造的真因の数式による確認

```
pred_lo = size_avg_pred - MAE (中央 1.75cm)
actual_min = actual_avg - actual_range/2 (中央 7.00cm)

pred_lo - actual_min
= (actual_avg - 1.75) - (actual_avg - 7.00)
= 7.00 - 1.75
= +5.25 cm  (実測中央値 +4.90cm と一致)
```

actual_range/2（7.0cm）が MAE（1.75cm）の **4.0 倍** という構造的差異が根本原因。

**結論: wMAPE が良いのに promise_break が悪い理由は、size のレンジ生成を「±MAE で対称」としていることで、pred_lo が actual_min をほぼ必ず上回るため。仮説（size_avg = (size_min + size_max)/2 の中央値張り付き構造）を実データで確認した。**

## 7. size 用 防御策の倍率候補

### size_min / size_avg の詳細分布（n=52,047 件）

| パーセンタイル | P5 | P10 | P25 | P50 | P75 | P90 | P95 |
|---|---|---|---|---|---|---|---|
| size_min / size_avg | 0.581 | 0.634 | 0.690 | **0.750** | 0.802 | 0.870 | 0.923 |
| size_max / size_avg | 1.077 | 1.130 | 1.198 | **1.250** | 1.310 | 1.366 | 1.420 |
| range / size_avg | 0.154 | 0.261 | 0.396 | **0.500** | 0.621 | 0.732 | 0.839 |

### cnt との比較

| | cnt P10 | cnt P50 | cnt P90 |
|---|---|---|---|
| _min / avg | 0.000 | 0.280 | 0.852 |
| _max / avg | 1.231 | 1.782 | 2.250 |

### 防御策推奨値

**防御策①（floor: pred_lo の下限）:**

cnt では `pred_lo >= avg_cnt × 0.3`（P50 = 0.280 付近）を採用した。
size は P10 = 0.634、P25 = 0.690 で **cnt より遥かに高い**。
ただし独立モデルが学習した size_min は avg_size × 0.70-0.75 付近を予測するはずであり、
安全マージンを見て **`pred_lo >= avg_size × 0.55`** を推奨。
- 根拠：P5 = 0.581 が実際の min/avg 最低値。モデルが P5 以下を予測した場合のみ floor 発動。
- 発動率（推定）：size_lo モデル出力の P5 以下 ≈ 5-10% のコンボで発動

**防御策②（clamp: レンジ幅の上限）:**

actual range / avg_size の P90 = 0.732。
cnt の clamp は avg_cnt × 2.0（P75 ≈ 2.0 - 1.5 = 0.5）だったが、
size は P90 range ratio = 0.732 なので **`(pred_hi - pred_lo) <= avg_size × 1.0`** を推奨。
- 根拠：P95 = 0.839 のレンジでも許容。1.0 は上位 5% 以外をカバー。
- 発動率（推定）：ほぼ発動せず（独立モデルが実際の range 1.0 を超える予測はまれ）

| 防御策 | 推奨値 | 発動率目安 | cnt 対比 |
|--------|--------|-----------|---------|
| floor: pred_lo >= avg_size × X | **0.55** | ~10% | cnt は 0.3（P50=0.280） |
| clamp: range <= avg_size × X | **1.0** | ~5% | cnt は 2.0（P90=2.25-1.0=1.25） |

## 結論

### Phase B 拡張で size_min/size_max 独立モデル化は効果的か

**判定: 効果あり（高）**

理由：
1. **promise_break 53% の原因は特定済み**：pred_lo = size_avg - MAE（1.75cm）が actual_min = actual_avg - 7.0cm より系統的に +4.9cm 高い（90.3% のケースで）
2. **wMAPE は良いが誤解を招く指標**：size_avg の center 精度は高いが、size_min の予測精度は別問題
3. **独立モデルなら actual_size_min を直接ターゲット**にできるため、バイアスを構造的に解消可能
4. **size_min/size_max の相関 r=0.94 は高いが**、OLS は識別可能。cnt_min/max も同種の高相関で実装済み

### 推奨される size 用 防御策値

- floor: `pred_lo >= avg_size × 0.55`
- clamp: `(pred_hi - pred_lo) <= avg_size × 1.0`

※ cnt の floor=0.3 / clamp=2.0 と全く異なる倍率。size を cnt と同じ倍率で扱うと過大なクリップが発生するため注意。

### Phase B 実装着手の Go/NoGo

**判定: Go**

根拠：
- 問題の真因が数式で特定でき、実データ（195件）で検証済み
- 独立モデル化は cnt で既に実装済みのパターンの流用であり技術的リスクは低い
- size_min / size_max の r=0.94 は多重共線性リスクだが、CNT で許容済みの同種問題
- 防御策の倍率は実データから定量的に導出済み
- wMAPE（現行 7.3% P50）は改善しないか若干悪化する可能性があるが、**primary KPI は promise_break であり、こちらの改善が目標**
- 全コンボ再実行で size promise_break が 53% → 30% 以下になれば商品品質に直結
