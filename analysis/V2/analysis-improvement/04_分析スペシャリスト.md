## 分析の開発ルール（CLAUDE.mdより）

1. **推測でjoinしない** — 実データで粒度を確認
2. **粒度の違うデータを混ぜない** — 日次と旬別を安易に混ぜない
3. **ループ実行前に `print(f'対象: {len(list)}件 先頭3: {list[:3]}')` で確認**
4. **JSONのキー取得はBashで直接確認** — エージェント要約を信頼しない

---

## 判断基準

| 方針 | 内容 |
|------|------|
| データ粒度 | 船宿×魚種×旬（combo_decadal）がベースライン |
| サンプル数 | 最低30件。15件未満は分析対象外と明示 |
| 欠航除外 | is_cancellation=1 は必ず除外 |
| ポイント座標 | lat/lon がNULLの行の扱いを明示 |
| 変数の意味 | 不明な変数は船長に確認してから使う |

---

## 品質ルール

1. **combo_deep_dive.py実行前にサンプル数を確認** — 魚種×船宿の組み合わせ件数
2. **分析結果は必ずanalysis.sqliteに保存** — テキストファイルだけで終わらせない
3. **バックテスト（backtest.py / backtest_oos.py）で精度を検証** — 主観的な「良くなった」は禁止
4. **_paths.pyのインポートパターンを守る** — ROOT_DIR, DATA_DIR, RESULTS_DIR
