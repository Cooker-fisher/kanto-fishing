## ship_trip_slot_map.json 設計

新規ファイル `normalize/ship_trip_slot_map.json` を導入。船宿ごとに `trip_no → time_slot` のフォールバックマップを定義する。

```json
{
  "_comment": "船宿ごとの trip_no → time_slot マッピング。fish_raw/kanso_raw でスロット確定できない場合のフォールバック。trip_no=2 は曖昧なため含めない（保守的設計）。",
  "林遊船": {
    "1": "午前",
    "3": "午後",
    "4": "午後",
    "5": "午後",
    "6": "午後",
    "7": "午後"
  }
}
```

**設計原則**:
- trip_no=2 は曖昧（80%午前/19%午後）のため**保留**（含めない）
- ships.json で `exclude:true` / `boat_only:true` の船宿は本マップにも追加しない
- 既存の fish_raw/kanso_raw キーワード抽出が**最優先**。trip_no マッピングは**フォールバック**のみ
