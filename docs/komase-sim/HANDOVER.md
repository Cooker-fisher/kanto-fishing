# Komase Sim Handover

## UI / UX Redesign Constitution (v33)

- この文書をUI/UX改善の正本とし、モックと本番は情報階層・余白・配色・レスポンシブ挙動を一致させる。
- 対象はコマセsim本体画面のみ。サイトトップ、LP、導線ページは本PR対象外。
- 診断結果カードを入力パネルより上位に配置し、結果→理由→調整の順で読める構成を優先する。
- PCは「メイン表示 + 右設定パネル」の2カラム、スマホは1カラムに切り替える。
- 設定パネルはスマホで折りたたみ可能にし、初期状態は閉じる。
- 375px幅で横スクロールを発生させない。固定幅コンポーネントを禁止する。
- 文言は釣り人向けに統一する。
  - 自動最適化 → 釣れる設定を探す
  - 合否 → 食わせ位置判定
  - 付けエサ到達率 → 付けエサがコマセに入る確率
  - 運用ログ → 詳細ログ
  - 本線・ビシ → 道糸・ビシ
  - しゃくり・巻き → 撒き方・待ち方
- 原色、派手なグラデーション、絵文字多用、過剰な装飾は使わない。
- 小魚の三角表現は禁止し、単純形状での可視化に統一する。
- 物理計算・optimizer・simulateHeadless のロジックはUI改修で変更しない。
