# 釣果価値チェッカー — プロジェクト

## 目的

釣果の金額をレンジで表示する独立アプリ。

> 入力: 魚種・匹数・平均サイズcm or 平均重量kg
> 出力: 卸売換算レンジ・小売換算レンジ

第一フェーズは **「いくら分か」だけ** を出す。可食部・下処理時間・食卓換算・船代回収率はやらない。

---

## スコープ（Phase 1）

| 項目 | 含む / 含まない |
|---|---|
| 卸売換算レンジ | 含む |
| 小売換算レンジ | 含む |
| 推定総重量 | 含む（計算過程として）|
| 全魚種（docs/fish/ 配下65魚種）対応 | 含む |
| 卸値クロール体制 | 含む（設計のみ・実装は後続PR）|
| 倍率テーブル（卸→小売）| 含む |
| サイズ・重量による単価変動 | 含む |
| UI/UX設計 | 含む |
| ----- | ----- |
| 可食部・歩留まり計算 | 含まない |
| 下処理時間 | 含まない |
| 食卓換算（食数）| 含まない |
| 船代回収率 | 含まない |
| X投稿文生成 | 含まない（Phase 1で保留）|
| 既存ページ改修 | 含まない |
| 釣果速報ページとの連携 | 含まない |

Phase 1 で確定する3つの設計ドキュメント:

- `プロジェクト.md` — このファイル
- `データ.md` — 魚種対比表・卸値クロール体制・倍率設計・サイズ単価
- `デザイン.md` — UI/UX設計

---

## 独立性ルール（絶対）

このアプリは **`docs/fish-value/` 配下で自己完結** する。

| 項目 | ルール |
|---|---|
| フォルダ境界 | `docs/fish-value/` 配下のみ追加・編集する |
| 既存サイト改修 | しない（crawler.py / analysis/ / design/ / docs/fish/ 等を触らない）|
| 唯一の接点 | 魚種が `docs/fish/` に追加されたとき、`fish-species-map.json` を同期する（手動・接点1箇所）|
| デプロイ | 配信は GitHub Pages `/docs/fish-value/` で受ける。CNAME 変更なし |
| crawl.yml | **触らない**。卸値クロールは別 workflow `.github/workflows/wholesale.yml` を独立追加（Phase 1 で許容される唯一の `.github/` 例外）|

既存サイトと同居しているが、**サイトの一部ではない独立アプリ**として扱う。

参考: 既存の独立フォルダ `docs/komase-sim/`（コマセシミュ）と同じ運用。

---

## 後続フェーズ（Phase 1 の責務外）

Phase 1 完了後にスコープを再設計する。本ドキュメントでは「やる/やらない」だけ決め、内容は確定させない。

- Phase 2 候補: 可食部・下処理時間
- Phase 3 候補: 食卓換算・船代回収率
- Phase 4 候補: X投稿文生成
- Phase 5 候補: 釣果速報ページ「この釣果で試す」導線

各Phaseは独立PR・独立MDで設計する。スコープを Phase 1 に持ち込まない。
