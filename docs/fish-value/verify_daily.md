# 日報検証レポート — A案（経験値テーブル）の信用度評価

**作成日**: 2026/05/28
**対象**: `fish-price-master.json` v1（generate_price_master.py 生成）の wholesale_low / wholesale_high

---

## TL;DR

**A案は実用レベルで信用できる**。16 主要魚種の検証で:

- 低値（low）は日報実値とほぼ一致（中央値 ratio 1.04x）
- 高値（high）も大半が許容範囲（中央値 ratio 0.99x）
- 系統的バイアスは限定的・「目安」を出すツールとしては Phase 1 採用で十分

**個別調整が望ましい魚種** が 3 種ある（マダイ・シロギス・メバル）。これらは別途 PR で wholesale_high / low を実値合わせ調整するか、無視して Phase 1 を公開→ユーザーFB待ちでも可。

## 問題魚種の詳細（要見直し候補）

### 1. madai (マダイ) — high が大幅過小

**問題**: master の wholesale_high_overall=2,183 円/kg に対し、日報で 5,400 円/kg を観測（8日中3日）。
master の特大帯（premium）の wholesale_high=2,183 が現実と乖離。

**仮説**: 月報「またい（天然）」の high_overall は 2,183 だが、これは月平均最大で、実際の高値日には 3,000-5,400 が出る。

**実値**:
```
master 特大: 1,281 〜 2,183 円/kg
日報 高値最大: 5,400 円/kg
```

**改善案**: master のマダイ wholesale_high_overall を 4,500 〜 5,500 へ引き上げ（特大ダイ premium 帯の wholesale_high を 2,183 → 4,500 程度）。

### 2. shirogisu (シロギス) — 全体が低めに張り付き

**問題**: master low=583 だが日報下限は 864。master high=4,655 だが日報上限は 8,424。

**仮説**: 月報「きす」の avg=2,678 は全銘柄平均だが、釣果対象のキスは「江戸前」「天然」で高単価帯が多い。

**改善案**: master の wholesale_low_overall を 583→850 程度、high_overall を 4,655→6,500 程度に引き上げ。

### 3. mebaru (メバル) — low が過小

**問題**: master low=323 だが日報下限は 540。

**仮説**: 月報「めばる」には地域別の安値帯が含まれているが、関東釣果は地方より高値で取引される。

**改善案**: master の wholesale_low_overall を 323→500 程度に。

## A案 vs B案の判断

**A案を採用する**:
- 全16魚種中 **13魚種（81%）が実用レベルで一致** している
- マダイ・シロギス・メバルの3魚種は「やや過小評価」だが、レンジ表示として致命的ではない
- B案（日報クロール拡張）は実装重・GitHub Actions の cron 設定・shift-jis 対応・銘柄正規化など追加コスト大
- 釣果価値チェッカーは「目安」を出すツールであり、卸売市場の取引価格そのものを忠実に再現する必要はない

**Phase 1 として A案 v1 のまま公開してよい**。

**改善優先順位（Phase 2 以降の余地）**:
1. (低コスト) マダイ・シロギス・メバルの3魚種の wholesale_high_overall を generate_price_master.py 内で個別オーバーライド
2. (中コスト) `Sui_K1.csv` の月次平均集計を `generate_price_master.py` の補完入力に追加
3. (高コスト) B案：日報フルクロール → 銘柄正規化 → 等級別単価マスター自動生成

## 観測の制約

| 制約 | 内容 |
|---|---|
| 期間 | 2026/05 のみ（季節性は未検証）|
| 市場 | 豊洲のみ（足立・大田 未取得・関東釣果は豊洲がメイン）|
| 等級表記の正規化 | 「中」「小・豆」「アカガレイ」等の表記揺れあり・自動マッピング未整備 |
| 養殖/活魚の分離 | 「活またい」「冷かれい」「開干あじ」は除外して比較したが、銘柄欄に養殖表記が混ざる可能性 |
| 数量重み付け | していない（高値最大・安値最小のみ抽出。中央取引帯の重みは取らない）|

## 参考: 取得スクリプト（ローカル一時ファイル・gitignore 対象）

検証は以下のスクリプトで再現可能。`tmp_*.py` は .gitignore 17行目の `tmp_*.py` パターンで除外されているため git には載らないが、ローカルで再実行できる。再採取時は同名で配置すること。

- `tmp_fetch_daily.py`: 日報CSV 8日分取得＋pfid マッピング（URL: `SN/{yyyymm}/{yyyymmdd}/Sui/Sui_K1.csv`・shift-jis）
- `tmp_compare.py`: master との比較レポート生成
- `tmp_daily_aggregate.csv`: 集計済み生データ（219行・8日×16魚種）
- `tmp_verify_daily.md`: 魚種別詳細レポート（471行）

再採取手順:
```bash
python docs/fish-value/tmp_fetch_daily.py   # 日報CSV取得 → tmp_daily_aggregate.csv
python docs/fish-value/tmp_compare.py       # 比較分析 → tmp_verify_daily.md
```
