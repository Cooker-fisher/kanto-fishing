# 釣果価値チェッカー — デザイン

## 1. 基本方針

| 項目 | 方針 |
|---|---|
| トーン | 既存サイト（funatsuri-yoso.com）と同系統。紺・白・薄グレー・オレンジ系アクセント |
| 雰囲気 | 実用カード。釣果を振り返るためのツール |
| 禁止 | 派手なグラデーション・絵文字多用・金ピカ・カジノ風・ゲーム風UI・単一価格断定表示 |
| デバイス優先 | スマホ縦持ち（375px幅）優先・PC は二次 |
| ページ数 | 1ページ完結（タブなし・モーダルなし）|

派手にしないこと。釣果報告フォーマッターのような落ち着いた業務カードに揃える。

---

## 2. 画面構成

```
┌──────────────────────────┐
│ ヘッダー                 │ ロゴ + 「釣果価値チェッカー」
├──────────────────────────┤
│ 1. 入力エリア            │ 魚種 / 匹数 / サイズ or 重量
├──────────────────────────┤
│ 2. 結果カード            │ 卸売・小売・推定総重量
├──────────────────────────┤
│ 3. 計算根拠              │ どのサイズ帯・どの倍率を使ったか
├──────────────────────────┤
│ 4. 注意文                │ 価格は概算である旨
├──────────────────────────┤
│ フッター                 │ ホームに戻るリンク（唯一の接点）|
└──────────────────────────┘
```

タブ・サブページなし。1スクロールで完結する。

## 3. 入力エリア

### 3.1 入力項目

| # | 項目 | 形式 | 必須 |
|---|---|---|---|
| 1 | 魚種 | ドロップダウン（65件・サイト表示順）| 必須 |
| 2 | 匹数 | 数値入力（1〜999）| 必須 |
| 3a | 平均サイズ | 数値入力（cm）| `input_modes=cm` のとき必須 |
| 3b | 平均重量 | 数値入力（kg）| `input_modes=kg` のとき必須 |

3a / 3b は魚種選択で自動切替（魚種マスタの `input_modes` 駆動）。両方は出さない。

### 3.2 ドロップダウンの並び順

サイト `docs/index.html` の魚種カード表示順を参考にする。Phase 1 では以下の優先順:

1. よく釣れる順（アジ・マダイ・タチウオ・カワハギ・イサキ・シロギス・ヒラメ等）
2. 50音順（その他）

実装は `fish-species-map.json` の配列順をそのまま使う。

### 3.3 スマホ最適化

- 入力フィールドは縦積み（横並びにしない）
- フォントサイズ最小14px（拡大なしで読める）
- 数値入力は `inputmode="numeric"` で数字キーボードを出す
- ドロップダウンはネイティブ `<select>`（カスタム UI にしない）

## 4. 結果カード

### 4.1 表示優先順位

1. **卸売換算レンジ**（メイン・大きく表示）
2. **小売換算レンジ**（メイン・大きく表示）
3. 推定総重量（補足）
4. サイズ帯ラベル（小型/中型/大型）

### 4.2 レンジ表示の絶対ルール

価格は **必ずレンジ表示**。

| ダメな例 | 良い例 |
|---|---|
| アジ35匹 = 6,428円 | アジ35匹 = 小売 約6,700〜20,200円 |
| 1匹あたり183円 | 1匹あたり 約190〜580円 |

単一価格・小数点付きの数値は出さない。100円単位で丸める。

### 4.3 カードのレイアウト案

```
┌──────────────────────────┐
│ 小売換算                  │
│ 約 6,700 〜 20,200 円     │ ← 大きく
├──────────────────────────┤
│ 卸売換算                  │
│ 約 3,400 〜  6,700 円     │
├──────────────────────────┤
│ 推定総重量  5.6 kg        │
│ サイズ帯    中型          │
└──────────────────────────┘
```

色は紺ベース・数値はオレンジ系アクセント。背景は薄グレー。

---

## 5. 計算根拠セクション

ユーザーが「なぜこの価格になったか」を辿れるようにする。

```
計算根拠
- 平均サイズ 25cm → 推定重量 0.16kg/尾
- サイズ帯: 中型（0.10〜0.22kg）
- 卸値: 600〜1,200円/kg（出典: 豊洲市場日報 2026/05/19）
- 倍率: 卸→小売 2.0〜3.0倍（一般値）
```

折りたたみ可能（デフォルト閉）。テキストは小さく・薄グレー文字。

---

## 6. 注意文（必須）

結果カードの直下に必ず表示する。

> 市場価格・鮮度・産地・季節で大きく変動するため概算です。実際の取引価格を保証するものではありません。

文字色は薄グレー・サイズ12px。

---

## 7. URL パラメータ

ページの初期値を URL から受け取る。

```
/docs/fish-value/?fish=aji&count=35&size=23
/docs/fish-value/?fish=madai&count=1&weight=1.2
```

| パラメータ | 内容 | 例 |
|---|---|---|
| `fish` | `site_fish_id` | `aji` / `madai` / `bishi-aji` |
| `count` | 匹数 | `35` |
| `size` | 平均サイズcm（cm入力魚種のみ）| `23` |
| `weight` | 平均重量kg（kg入力魚種のみ）| `1.2` |

不正値はサイレントに無視（デフォルト値で起動）。

---

## 8. 色・タイポグラフィ

既存サイトの `docs/style.css` から色変数を参照（**コピーしない**）。`docs/fish-value/` 内で独自CSSを定義するが、色値は既存サイトと一致させる。

| 用途 | 色（参考値・既存サイト準拠）|
|---|---|
| 背景 | `#ffffff` |
| メインテキスト | `#0f1f3a`（紺）|
| サブテキスト | `#6b7280`（グレー）|
| アクセント | `#f97316`（オレンジ）|
| カード背景 | `#f3f4f6`（薄グレー）|
| ボーダー | `#e5e7eb` |

フォント: 既存サイトと同じ（システムフォント）。

## 9. レスポンシブ要件

| 幅 | レイアウト |
|---|---|
| 375px | 縦1列・入力フィールド縦積み・結果カード幅100% |
| 768px | 縦1列維持・最大幅640pxセンタリング |
| 1366px | 縦1列維持・最大幅640pxセンタリング |

PCでは横に広げない。常に縦長レイアウト。

横スクロール禁止（375px幅で発生したら CSS バグ扱い）。

---

## 10. 接点ルール（厳守）

| 接点 | ルール |
|---|---|
| トップページからのリンク | PR5 で1箇所追加。トップページ既存レイアウトは変更しない。リンクカードを1枚追加するのみ |
| 戻る導線 | フッターに「ホームに戻る」リンク1個のみ |
| クロスリンク | 魚種ページや釣果ページとの相互リンクは作らない（Phase 1 では）|
| 共通CSS | `docs/style.css` を読み込まない（独立CSS で完結）|
| 共通JS | `docs/main.js` を読み込まない（独立JS で完結）|

唯一の接点はトップページのリンクカード1個のみ。

## 11. アクセシビリティ最低要件

| 項目 | 要件 |
|---|---|
| キーボード操作 | Tab で全項目フォーカス可能 |
| スクリーンリーダー | `<label>` を全入力に紐付け |

| エラー表示 | 不正入力は赤テキストで即時表示（モーダル禁止）|

## 12. 実装制約

| 項目 | 制約 |
|---|---|
| フレームワーク | 使わない（vanilla JS）|
| ビルド | 不要（HTML/CSS/JS をそのまま配信）|
| 外部依存 | なし（CDN リンクも禁止）|
| ファイル数 | `index.html` + `app.js` + `style.css` の3つに絞る |

既存サイトと同じ「静的ファイル直配信」方針を維持する。
