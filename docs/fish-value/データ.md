# 釣果価値チェッカー — データ

このドキュメントは、Phase 1 で使うすべてのデータ設計を定義する。

- 魚種対比表（サイト表示名 ↔ 価格計算ID）
- 卸値クロール体制
- 倍率テーブル（卸値 → 小売値）
- サイズ・重量による単価変動

---

## 1. 魚種対比表

### 1.1 設計方針

サイト上の魚種ページ（`docs/fish/*.html`・65件）と、価格計算上の正規化魚種（`price_fish_id`）を **分離** する。

理由:

- サイト上では「アジ」「ビシアジ」を別釣り物として扱う（コマセ釣り vs LT釣り）が、価格計算上はどちらも「マアジ」として扱いたい
- 「イナダ」「ワラサ」「ブリ」は表示上は別だが、ブリ系のサイズ帯違いで内部処理したい
- 複合釣り物（例: ヤリイカ・スルメイカ）は価格計算上は主魚種に寄せたい

### 1.2 マスタファイル

```
docs/fish-value/fish-species-map.json
```

各エントリのスキーマ:

```jsonc
{
  "site_fish_id": "bishi-aji",           // docs/fish/{site_fish_id}.html と一致
  "site_display_name": "ビシアジ",       // サイトで表示する名前
  "price_fish_id": "maaji",              // 価格マスタ参照キー
  "aliases": ["ビシアジ", "コマセアジ"], // TSURI_MONO_MAP から転記
  "input_modes": ["count_avg_cm"],       // cm入力 or kg入力 or 両方
  "note": "魚価換算ではマアジとして扱う"
}
```

### 1.4 魚種追加時の手順（唯一の接点）

新魚種が `docs/fish/` に追加されたとき:

1. `fish-species-map.json` に新エントリを追加
2. 必要なら `fish-price-master.json` にも `price_fish_id` を追加
3. このMDの対比表に1行追記
4. アプリ側コード変更不要（JSON駆動）

これ以外の同期作業は発生しない。

## 2. 卸値クロール体制

### 2.1 ソース候補（一次情報）

| ソース | URL | 更新頻度 | 形式 |
|---|---|---|---|
| 東京都中央卸売市場 日報 | `https://www.shijou.metro.tokyo.lg.jp/torihiki/geppo/` | 日次 | CSV/PDF |
| 水産庁 産地市場日報 | `https://www.jafic.or.jp/sijou/` | 日次 | HTML/PDF |
| 全水卸（参考）| `https://www.zensuiren.or.jp/` | 月次 | PDF |

**Phase 1 採用**: 東京都中央卸売市場日報（豊洲）を主、他は将来。

### 2.2 クロール頻度

- **週1回**（毎週月曜 16:30 JST 想定）
- 日次更新は不要。市場価格は日変動するが、平均化を狙うため週1で十分
- 既存 `crawl.yml`（毎日16:30）とは別の workflow を立てる。crawl.yml は触らない

### 2.3 出力スキーマ

```
docs/fish-value/wholesale-prices.json
```

```jsonc
{
  "updated_at": "2026-05-20",
  "source_url": "https://www.shijou.metro.tokyo.lg.jp/torihiki/geppo/...",
  "prices": [
    {
      "price_fish_id": "maaji",
      "date": "2026-05-19",
      "size_band": "中型",
      "low_yen_per_kg": 600,
      "high_yen_per_kg": 1200,
      "sample_n": 12
    }
  ]
}
```

複数日分の値が貯まったら、`size_band` ごとに移動平均（直近4週）で代表値を出す。

### 2.4 クロール実装方針（Phase 1 確定・実装は PR3）

| 項目 | 方針 |
|---|---|
| 配置 | `docs/fish-value/crawl_wholesale.py` |
| 依存 | 標準ライブラリのみ（既存 `crawler.py` 方針に揃える）|
| リクエスト間隔 | 0.8秒（既存 `crawler.py` と同じ）|
| 実行環境 | GitHub Actions 別 workflow（`.github/workflows/wholesale.yml` を新規追加）|
| 失敗時の挙動 | 既存 `wholesale-prices.json` を保持（クラッシュさせない）|

### 2.5 PDF パース戦略

東京都市場日報は PDF が主体。Phase 1 では PDF パースは導入せず、**CSV版がある魚種のみ** を対象とする。PDFしかない魚種は手動入力で `wholesale-prices.json` に追記する運用とする（Phase 1 ではエッジケース扱い）。

## 3. 倍率テーブル（卸値 → 小売値）

### 3.1 設計方針

魚種ごとに `retail_multiplier_low` / `retail_multiplier_high` を持つ。

```
小売換算下限 = 卸売換算下限 × retail_multiplier_low
小売換算上限 = 卸売換算上限 × retail_multiplier_high
```

### 3.2 倍率レンジ（初期値・出典: 水産流通の一般値）

| 価格帯 | 例 | retail_multiplier_low | retail_multiplier_high |
|---|---|---|---|
| 大衆魚 | アジ・サバ・イワシ・キス | 2.0 | 3.0 |
| 中級魚 | マダイ・カワハギ・ヒラメ・イサキ | 1.8 | 2.5 |
| 高級魚 | アカムツ・クロムツ・キンメ・トラフグ | 1.6 | 2.2 |
| 青物 | カツオ・ワラサ・ブリ・キハダ | 1.7 | 2.4 |
| イカ・タコ | スルメイカ・ヤリイカ・マダコ | 1.8 | 2.6 |

倍率は `fish-price-master.json` の各 `price_fish_id` 直下に持つ。価格帯テーブルではなく、魚種ごとに直書きする（修正容易性のため）。

### 3.3 出典の明記

`fish-price-master.json` の各エントリに `source_note` を必須にする。

```jsonc
{
  "price_fish_id": "maaji",
  "retail_multiplier_low": 2.0,
  "retail_multiplier_high": 3.0,
  "source_note": "豊洲卸 → スーパー小売の典型的なマージン。流通段階で2〜3倍が一般値（一次資料: 農水省 食料流通統計）"
}
```

将来、出典の見直しが必要なときに辿れるようにする。

## 4. サイズ・重量による単価変動

### 4.1 設計方針

同じ魚種でも、サイズが違えば単価が大きく変わる（小アジと大アジ、中ダイと大ダイ）。

各魚種に以下を持つ:

- `size_weight_curve`: cm → kg の推定曲線（離散点の配列）
- `price_bands`: kg帯ごとの卸値レンジ

### 4.2 cm入力時の3段フロー

```
[入力] 平均サイズ 25cm × 35匹
   ↓
[1] size_weight_curve で 25cm → 0.16kg/尾
   ↓
[2] price_bands で 0.16kg → 「中型」帯 → 卸 600〜1200円/kg
   ↓
[3] 推定総重量 5.6kg × 卸600〜1200円/kg = 卸 3,360〜6,720円
              × 倍率2.0〜3.0 = 小売 6,720〜20,160円
```

### 4.3 size_weight_curve の構造

```jsonc
{
  "price_fish_id": "maaji",
  "size_weight_curve": [
    { "cm": 15, "kg": 0.05 },
    { "cm": 20, "kg": 0.10 },
    { "cm": 25, "kg": 0.16 },
    { "cm": 30, "kg": 0.25 },
    { "cm": 35, "kg": 0.38 }
  ]
}
```

入力cm値が中間値なら線形補間する。

### 4.4 price_bands の構造

```jsonc
{
  "price_fish_id": "maaji",
  "price_bands": [
    {
      "label": "小型",
      "min_kg": 0.03,
      "max_kg": 0.10,
      "wholesale_low_yen_per_kg": 400,
      "wholesale_high_yen_per_kg": 800
    },
    {
      "label": "中型",
      "min_kg": 0.10,
      "max_kg": 0.22,
      "wholesale_low_yen_per_kg": 600,
      "wholesale_high_yen_per_kg": 1200
    },
    {
      "label": "大型",
      "min_kg": 0.22,
      "max_kg": 0.60,
      "wholesale_low_yen_per_kg": 900,
      "wholesale_high_yen_per_kg": 1800
    }
  ]
}
```

`wholesale_prices.json` が存在する場合は **そちらを優先** し、price_bands の値は fallback として使う。

### 4.5 kg入力時のフロー

cm→kg 変換ステップをスキップし、直接 price_bands でサイズ帯判定する。

```
[入力] 平均重量 1.2kg × 1匹
   ↓
[1] price_bands で 1.2kg → 「大型」帯
   ↓
[2] 推定総重量 1.2kg × 卸 1800〜3500円/kg = 卸 2,160〜4,200円
              × 倍率1.8〜2.5 = 小売 3,888〜10,500円
```

### 4.6 入力モードの選択

`fish-species-map.json` の `input_modes` で魚種ごとに固定する:

- `cm` のみ: 小型魚（アジ・イサキ・カワハギ・シロギス等）
- `kg` のみ: 大型魚（マダイ・ヒラメ・青物・高級魚）
- `cm, kg` 両方: 一部魚種は将来検討（Phase 1 はどちらか一方に固定）

両方選択肢を出すと UI が複雑化するため、Phase 1 は1魚種=1モードに絞る。

## 6. データ更新フロー

```
[週1] crawl_wholesale.py が東京都市場日報をクロール
   ↓
wholesale-prices.json 更新
   ↓
[ユーザー操作] index.html が wholesale-prices.json を fetch
   ↓
無ければ fish-price-master.json の price_bands を fallback として使用
   ↓
画面に卸売・小売レンジを表示
```

## 7. データ品質ルール

| ルール | 内容 |
|---|---|
| 単一価格断定禁止 | 必ずレンジ（low〜high）で持つ |
| 出典明記 | 各エントリに `source_note` |
| 更新日明記 | 各JSONに `updated_at` |
| 推測値の明示 | 一次データが無い場合は `note: "推測値"` を付ける |
| 倍率の極端値禁止 | `retail_multiplier_high / low ≤ 2.0` を守る（不自然な広レンジ防止）|
