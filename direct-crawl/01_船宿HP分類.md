# 01_船宿HP分類 — 5タイプ定義

> **変更ガバナンス**: catches_raw.json / data/*.csv のフィールド変更はオーナー最終承認 + 関連MD責任者了解が必須。

## タイプ一覧

| タイプ | 名称 | 代表船宿 | パーサー難度 | 情報量 |
|--------|------|---------|------------|--------|
| Type A | シンプル表型 | 敷嶋丸・啓秀丸 | 低 | 中〜低 |
| Type B | リッチ列型 | 山本釣船店（kurodai.com） | 高 | 最大 |
| Type C | ナラティブ型 | たいぞう丸 | 高 | 大 |
| Type E | 便別データのみ型 | 忠彦丸 | 最低 | 中 |

※ Type D は今回の調査対象に該当なし。将来の拡張用に欠番。

---

## Type A: シンプル表型

**代表**: 敷嶋丸、啓秀丸

### 特徴
- 4列テーブル: 釣りもの | 大きさ | 匹数 | 詳細
- 【ラベル】形式でセクション区切り（例: 【午前船】【午後船】）
- HTMLが単純なtable構造

### パーサー方針
- `<table>` パースで取得可能
- 【ラベル】から便(trip_no)を判定
- 啓秀丸は匹数「-----」のため数値抽出不可 → 対象外

### 取得可能フィールド
- fish_raw, size_raw, count_raw（敷嶋丸のみ）, suion_raw（水温あり）

---

## Type B: リッチ列型

**代表**: 山本釣船店（kurodai.com）

### 特徴
- 複数列テーブル: 日付 | 天気 | 水温 | 透明度 | 竿頭 | 個人別釣果
- タックル情報あり（エサ・ハリス・ハリ）
- 黒鯛専門で個人別の詳細データ
- 1日分が1行ではなく複数行にわたる

### パーサー方針
- 列マッピングが船宿固有 → 汎用化困難
- 個人別釣果は配列として格納（individual_catches）
- タックル情報はtackle_rawに格納

### 取得可能フィールド
- fish_raw, count_raw, size_raw, suion_raw, suishoku_raw（透明度）
- weather_raw（天気）, tackle_raw, top_catch_raw, individual_catches

### 注意
- catches_rawでは4,228件あるが平均16文字/件。HPには数倍の情報量がある

---

## Type C: ナラティブ型

**代表**: たいぞう丸（taizoumaru.com）

### 特徴
- メイン魚種 + サブ魚種列挙
- 長文コメント（船長の感想・状況説明）
- 写真複数枚
- 潮汐グラフ画像

### パーサー方針
- 構造化されていない部分が多い → 正規表現ベースの抽出
- 水温は「水温14.0℃」のようなパターンで抽出
- 潮汐は「【中潮】」パターン
- 写真URLは`<img>`タグから収集

### 取得可能フィールド
- fish_raw, count_raw, size_raw, kanso_raw（長文）
- suion_raw, tide_type_raw, photo_urls

### 注意
- 釣りビジョンにも詳細投稿する優良船宿のため、コメント内容の重複あり
- 水温・潮汐・写真のみが自社HP独自価値

---

## Type E: 便別データのみ型

**代表**: 忠彦丸（tadahikomaru.jp）

### 特徴
- 午前/午後/魚種別のセクション分け
- 数値のみ（コメント無し）
- 「(2隻高低)」表記で複数艇の合算データ
- 構造が最も機械的

### パーサー方針
- セクションヘッダから便(trip_no)と魚種(fish_raw)を判定
- 匹数はmin〜maxパターンで抽出
- 「(N隻高低)」からboat_countを抽出
- コメント(kanso_raw)は空文字

### 取得可能フィールド
- fish_raw, count_raw, boat_count
- time_slot（午前/午後）

### 注意
- **釣りビジョン未掲載（catches_raw 0件）** → 自社HPが唯一のデータソース
- boat_countの扱いが既存パイプラインに影響（06_データ定義.md参照）

---

## パーサー開発の優先順位

| 順位 | タイプ | 理由 |
|------|--------|------|
| 1 | Type E | パース最易 + データ不在エリア（金沢八景）のカバー |
| 2 | Type A | シンプル構造 + 水温取得 |
| 3 | Type B | 情報量最大だがパーサー複雑 |
| 4 | Type C | ナラティブ解析が必要 + 釣りビジョンで大部分カバー済み |
