### 既存フィールド活用（null → 値が入る）

- **suion_raw**: null → "16.0" 等（水温）。下流で null チェック済みのため影響低
- **suishoku_raw**: null → "薄濁" 等（透明度）。同上

---

## 互換性ルール（5原則）

1. **既存フィールドの型・形式は変えない**
2. **null だったフィールド（suion_raw / suishoku_raw）に値を入れるのは OK** — 下流で null チェック済み
3. **新規フィールドは全て optional** — 既存レコードに存在しなくてもエラーにならない
4. **source フィールドで出所を識別** — 分析時に釣りビジョン経由と自社HP直接を区別可能
5. **generate_csv_all() への影響** — 新フィールドは CSV に列追加するか無視するか選択。既存列は削らない・型を変えない

## boat_count の影響（最もリスクが高い変更）

### 問題

忠彦丸の「午前ライトアジ船(2隻高低) 8～61匹」は **2隻分の合算**。
現在の catches_raw は 1船宿=1隻前提で匹数を格納している。

### 影響範囲

| 層 | 影響 |
|-----|------|
| B (CSV) | cnt_min/max/avg の解釈が変わる |
| C (分析) | combo_decadal のベースラインに2隻合算値が混入 |
| D (予測) | 予測精度の劣化（1隻と2隻を同列比較） |
| E (表示) | ランキング表示で2隻船宿が有利に見える |

### 対策

- `boat_count` は必須フィールドとして追加すべき
- 既存レコードには `boat_count: 1`（デフォルト）を暗黙設定
- 分析時は `boat_count` で正規化するロジックを追加

## 変更ガバナンス手順

フィールド追加・変更時のプロセス:

1. 本ファイル（06_データ定義.md）に候補を追記
2. 07_パイプライン影響分析.md で影響範囲を評価
3. PIPELINE.md の変更インパクトマトリクスを更新
4. **オーナーの最終承認を取得**
5. **関連MD責任者（PIPELINE.md 管理者、redesign/ 責任者）の了解を取得**
6. 実装・テスト・デプロイ

**このプロセスに例外はない。**
