> **変更ガバナンス**: catches_raw.json / data/*.csv のフィールド変更はオーナー最終承認 + 関連MD責任者了解が必須。

## 基本情報

| 項目 | 内容 |
|------|------|
| 船宿名 | 敷嶋丸 |
| エリア | 大原港 |
| URL | shikishimamaru.com 相当 |
| HPタイプ | Type A（シンプル表型） |
| 情報ロス率 | 約20% |

## HP構造

- 4列テーブル: 釣りもの | 大きさ | 匹数 | 詳細
- 【ラベル】形式でセクション区切り（午前船/午後船等）
- HTMLが単純な `<table>` 構造

## 取得可能フィールド

| フィールド | 取得可否 | 例 |
|-----------|---------|-----|
| fish_raw | ✅ | マダイ等 |
| size_raw | ✅ | 30〜50cm |
| count_raw | ✅ | 2〜8匹 |
| suion_raw | ✅ | 14.5℃ |
| kanso_raw | ✅ | 詳細コメント |

## catches_raw.json との差分

- **水温（suion_raw）**: 自社HPには 14.5℃ 等の記載あり。catches_raw では全件 null
- その他のフィールドは釣りビジョン経由でほぼカバー済み

## 判定

**釣りビジョンで十分。** 水温のみ追加価値があるが、情報ロス20%のため優先度は低い。Phase 4以降で検討。
