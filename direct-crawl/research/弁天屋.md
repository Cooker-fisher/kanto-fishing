## 取得できる情報（突出して豊富）

| 情報 | フィールド | 備考 |
|------|-----------|------|
| 日付 | `date` | 一覧タイトルから "2026年4月6日" |
| 潮汐区分 | `kanso_raw` or 別活用 | "中潮"/"大潮"等 ← tide_moon.sqlite補完不要 |
| 魚種名 | `fish_raw` | 【】内から |
| サイズ | `size_raw` | "16〜25cm" or "0.4kg" |
| 数量 | `count_raw` | "91〜221尾" |
| 天気 | `kanso_raw`に付記 | "晴れ" |
| 風向 | `kanso_raw`に付記 | "北～南西" |
| 波の状況 | `kanso_raw`に付記 | "低い" |
| 船長コメント | `kanso_raw` | 詳細テキスト |

**天気・風向・波はweather_cacheの補完・検証に使える。**

## catches_raw フィールドへのマッピング

| bentenya.com の情報 | フィールド | 処理 |
|--------------------|-----------|------|
| 一覧タイトルの日付 | `date` | "2026年4月6日" → YYYY/MM/DD |
| 【魚種名】 | `fish_raw` | 【】を除去 |
| サイズ行 | `size_raw` | "16〜25cm" |
| 数量行 | `count_raw` | "91〜221尾" |
| 天気・風向・波 + 船長コメント | `kanso_raw` | まとめてフリーテキスト |
| 潮汐区分（"中潮"等） | `kanso_raw`に付記 | tide_typeはCSV層で抽出 |

## パーサー設計

1. `chouka.html` から全 `(no, date, tide_type)` を収集
2. 各 `detail.html?no=N` を取得
3. `【魚種名】` でセクション分割
4. プレースホルダー判定: `～cm ～尾` + `天気：～` → スキップ
5. 各フィールドを正規表現で抽出
