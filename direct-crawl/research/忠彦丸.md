## URLパターン

```
最新ページ:
  https://www.gyo.ne.jp/rep_tsuri_view%7CCID-tadahiko.htm

履歴ページ（過去6日分のみ有効）:
  https://www.gyo.ne.jp/rep_tsuri_history_view%7CCID-tadahiko%7Chdt-{YYYY/MM/DD}%7Cdt-{今日}.htm
  ├── hdt = 取得したい日付
  └── dt  = アクセス当日の日付（必須。7日以上前を指定すると最新ページにリダイレクト）
```

**検証済み:**
- `hdt=2026/03/01, dt=2026/04/06`（36日前） → 最新ページへリダイレクト
- `hdt=2026/04/01, dt=2026/04/06`（5日前） → 正常表示

**→ 遡れるのは当日 + 過去6日 = 計7日分のみ。過去データは取得不可。**

## パーサー設計上の難しい点

### ① 釣果日付フォーマットが不統一・typoあり

```
"４月６日釣果"       ← 月日あり（最多）
"6日釣果"           ← 日のみ（月なし）
"4月tadahiko1日釣果" ← typo（HTML固有の誤記）
"月日釣果"          ← プレースホルダー → スキップ
```
→ 年はページURL日付から補完。「月日釣果」はスキップ判定必須。

### ② 釣果日付 ≠ ページ日付

更新頻度が低い釣り物（ウィリー五目、タチウオ＆LTアジ等）は古いまま。
- ページ日付 2026/04/01 なのに釣果は「3月21日釣果」のまま
- ページ日付で上書き禁止。釣果行の日付を正しくパースして使う。

### ③ 複数隻・複数便が1セルに混在

```
"0.9～3.4kg ０～２匹(乗合船と仕立船２隻の高低)"  → 複数隻の高低
"午前⇒17～28cm １５～９０匹(２隻高低)"           → 複数隻
"午後⇒～cm ～匹"                                 → 未更新混在
```
→ `cnt_min`/`cnt_max` は高低からそのまま取る。`kanso_raw` に "(２隻高低)" 等を残す。

### ④ 釣り物セクション識別（テーブルがフラットに並ぶ）

全 `<table>` が同階層。船種ヘッダーの判定ロジック:
- 「乗合船」「仕立船」「限定」を含む → 船種ヘッダー（次のセクション開始）
- 「出船時間」「納竿時間」を含む → タックルテーブル（スキップ）
- 「日釣果」または「月日釣果」を含む → 釣果テーブル
- 「コメント」を含む → コメントテーブル（kanso_rawに格納）

### ⑤ 過去データ取得不可（7日制限）

新規導入しても 2023〜2026/04/05 の過去データは取れない。
**毎日クロールで蓄積するしかない → 早期開始が価値を高める。**
