## リデザインに向けた技術メモ

### テンプレート分離のしやすさ

**現状**: HTML生成は全て Python f-string によるインライン生成。テンプレートエンジン（Jinja2等）は未使用。

**分離の容易さ**: ★★☆☆☆（やや難しい）

理由:
1. **ロジックとHTMLが密結合**: build_html() は L4849-5104（約260行）で、データ加工とHTML組み立てが混在している
2. **CSS が5箇所に分散**: index/fish/area/fish_area/calendar/forecast で別々のCSS文字列。大部分は共通だが微妙に異なる
3. **ヘッダ・フッタが関数化されていない**: 各ページで個別にコピペ（forecast系のみ `_forecast_page_head/foot` で共通化）
4. **JSがインライン**: 外部JSファイルがない（CDNのChart.jsを除く）

**リデザイン時の推奨アプローチ**:
1. 共通CSS を1つの外部ファイル（`style.css`）に統合し、ページ固有の差分だけインラインまたは追加クラスで対応
2. 共通ヘッダ・フッタ・ナビを関数化（`_page_head()`, `_page_foot()`, `_page_nav()`）
3. JS を `main.js` に外出し（filter/sort/tab切替/GA追跡）
4. Jinja2 導入は crawler.py の「標準ライブラリのみ」制約と衝突するため非推奨。f-string のままで構造化する方が現実的
