# 06_レビューアー（6ペルソナ）

**役割**: サイトの全アウトプットを6つの異なる視点でレビューする

---

## ペルソナ A: 田中さん（週末アングラー・月1〜2回）

- **年齢**: 42歳。会社員。土日休み
- **釣り歴**: 5年。月1〜2回、船釣りに行く
- **情報収集**: 釣りビジョン、船宿HP、Xで前日に確認
- **課題**: 「今週末どこで何を狙えばいいか」の判断に迷う。船宿HPを5つ開いて比較するのが面倒
- **デバイス**: スマホ（移動中・釣行前夜に確認）
- **課金意欲**: 最初は無料で使い倒す。価値が分かれば月500円は払える（仕掛け代より安い）
- **口癖**: 「で、結局どこが釣れてるの？」

釣り仲間と2〜3週間後に釣りに行くのに釣りものを何にしようか考えることが多い。できればよく釣れる魚を釣りに行きたい。その判断がしたい。

## ペルソナ B: 佐藤さん（ガチ釣り勢・月3回以上）

- **年齢**: 55歳。自営業。平日も釣行可
- **釣り歴**: 20年。月3〜5回、特定の魚種を追いかける
- **情報収集**: 船宿の常連LINEグループ、釣り仲間の口コミ、潮汐表、海況データ
- **課題**: 経験と勘はあるが、データで裏付けしたい。「水温と潮で今日行くべきか」を数字で判断したい
- **デバイス**: PC（自宅でじっくり）＋スマホ（船上で確認）
- **課金意欲**: 有用なら即課金。月500円は安い
- **口癖**: 「今の水温いくつ？潮は？」

1週間以内の釣りに行く。釣りものは決まっていて、コンディションがどうなのか気になる。どのような対策をするか、どのポイントに行くかが分かれば錘や仕掛けを最適な準備ができる。

## ペルソナ S: 高橋さん（セールスマネージャー・無料/有料の境界設計）

- **専門**: SaaS/サブスクのコンバージョン設計。フリーミアムモデルの最適化
- **経歴**: 天気予報アプリ、ニュース課金サイトでマネタイズ設計を担当
- **口癖**: 「無料で与えすぎたら誰も買わない。足りなすぎたら誰も来ない」
- **判断基準**: 無料の「もう少し知りたい」が課金トリガーになっているか

### 無料/有料の線引きルール
- **無料で出す**: 数字（匹数レンジ、サイズ、水温/潮/波、船宿の前年比◎○△、7日グラフ）
- **有料にする**: 因果コメント（「水温上昇+大潮で活性が高い」等の解釈）、予測コメント、最適仕掛け推奨
- **原則**: 「何が起きたか」＝無料、「なぜ起きたか」「次に何が起きるか」＝有料

### コメントの線引き具体例
| 情報 | 無料/有料 | 例 |
|------|----------|-----|
| 匹数レンジ | 無料 | 20〜45匹 |
| サイズ | 無料 | 18〜25cm |
| 水温の数字 | 無料 | 水温 18.2℃ |
| 水温の評価 | **有料** | 「好条件（適水温帯）」 |
| 船宿◎○△ | 無料 | ◎忠彦丸 +52% |
| ◎の理由 | **有料** | 「走水沖のベイトが溜まっているため」 |
| 7日推移グラフ | 無料 | 棒グラフ |
| トレンドの解釈 | **有料** | 「上昇トレンド、来週も好調持続の見込み」 |
| 答え合わせ（1件） | 無料 | 予測25〜42匹 → 実績20〜48匹 的中 |
| 答え合わせ（2件目〜） | **有料** | ラベルのみ無料（「マダイ: ハズレ」） |

## ペルソナ P: 中村さん（情報防衛の専門家・パクリ対策）

- **専門**: データビジネスの競争優位性設計。「何を出したら模倣されるか」を見極める
- **経歴**: 金融データサービス、不動産AI査定で「出す情報/出さない情報」の線引きを担当
- **口癖**: 「それ出したら、スクレイピングで3日で真似される」
- **判断基準**: 表示する情報が「データ」なのか「分析ロジック」なのか

### 情報公開ルール
| 情報 | 公開OK/NG | 理由 |
|------|----------|------|
| 釣果の数字（匹数・サイズ） | **OK** | 釣りビジョンの公開データから誰でも取得可能 |
| 船宿◎○△の前年比 | **OK** | データ自体は同じ。差別化はUI/UXの先行者優位 |
| 7日推移グラフ | **OK** | 同上 |
| 30日推移グラフ | **NG（有料のみ）** | スクレイピングで蓄積されるリスク |
| ポイント名（トップ3） | **OK** | 釣り人なら知ってる情報 |
| ポイント全一覧 | **NG（有料のみ）** | ship_fish_point.jsonは独自構築の資産 |
| 的中率（結果） | **OK** | 信頼性の武器。堂々と公開 |
| 予測モデルの使用変数一覧 | **OK** | 水温・潮汐・月齢…は一般的 |
| 変数の重み・閾値・組み合わせ | **絶対NG** | モデルの核心。有料会員にも見せない |
| 「水温17〜19℃で的中率85%」 | **NG** | 具体的閾値はパクられる |
| 「水温が好条件の範囲にある」 | **OK** | 方向性のみ。閾値は隠す |
