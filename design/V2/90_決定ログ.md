### 2026/04/12 — combo_deep_dive 並列実行を標準とする

- **決定**: 全魚種 deep dive 実行は `run_full_deepdive.py --workers 4` を使う。逐次実行は禁止
- **理由**: 42〜55魚種を逐次実行すると30〜40分かかる。workers=4 並列で約1/4に短縮（SQLite WAL モードにより同時書き込みは安全にシリアライズ）
- **実行コマンド**:
  ```bash
  # 標準（全魚種 workers=4）
  python analysis/V2/methods/run_full_deepdive.py --workers 4

  # 特定魚種のみ再実行
  python analysis/V2/methods/run_full_deepdive.py アジ マダイ --workers 2

  # GitHub Actions（2CPU環境）
  python analysis/V2/methods/run_full_deepdive.py --workers 2
  ```
- **禁止パターン**: `for fish in fish_list: subprocess.run(...)` のような手書きループ。run_full_deepdive.py を経由すること
- **workers 推奨値**: ローカルSSD=4〜6 / GitHub Actions(2CPU)=2〜3 / HDD=2

### 2026/04/15 — Claude Code モデルフォーメーション

- **決定**: T1〜T7 など通常の実装・調査・コミット作業は **Sonnet 4.6** をメインモデルとして使用する。WebSearch や大量の参照情報の事実抽出が必要な場面は **Haiku 4.5 の subagent** を `Agent` ツールで呼び分離する。**Opus 4.6** は複数トピック横断の方針立案・大規模リファクタ設計など深い推論が要る場面に限定する

  - メインセッション（Sonnet 4.6）＝ JSON 整形、スキーマ適合編集判断、git commit／push、ユーザー対話
  - 調査 subagent（Haiku 4.5）＝ WebSearch 実行、エリア別タックル・釣法など事実の抽出、構造化結果で返却
  - Opus 4.6 ＝ 方針転換・アーキテクチャ変更・複数ファイル跨ぎの大規模設計のみ
- **理由**: 事実抽出と JSON 整形は Opus の深い推論を必要とせず、Sonnet でほぼ同品質。Haiku に調査を分離することで、生の検索結果を読むトークン消費をメインセッションから切り離せる。コスト概算で Opus → Sonnet＋Haiku で 5〜10 倍削減
- **切替手順**: Claude Code で `/model` → Sonnet 4.6 を選択。subagent は `Agent({ subagent_type: "general-purpose", model: "haiku", prompt: "..." })` で呼び出す
- **関連ロール**: 全ロール
