## 現状サイトの課題

ユーザーから「ごちゃごちゃしている」というフィードバックあり。

### 具体的な問題点
1. **セクション過多**: トップページに7セクション（狙い目・リスク・海況・予測・コンボ・急落・釣果）
2. **情報の重複感**: 「狙い目」と「釣れている魚」に同じ魚種が並ぶ
3. **カード内の情報密度が高すぎる**: ★評価+前年比+理由タグ3つ+期待度バー+コメント
4. **空セクション**: 釣果予測が未実装で中途半端に表示
5. **類似セクションの分散**: 「注目コンボ」と「急落コンボ」が別セクション

## デザイン原則

### 判断を委任されている項目
- 配色テーマ（現状ダークネイビー → 変更可）
- ナビゲーション構造
- レイアウト全般

### 制約
- レスポンシブ対応必須（モバイル/PC両方同等）
- GitHub Pagesでの静的ホスティング
- HTMLはcrawler.pyが自動生成（テンプレートエンジンなし）

### AdSense審査対応
審査が現在却下中。デザイン面で対応が必要：
- オリジナルコンテンツの見せ方（スクレイピングデータの付加価値化）
- プライバシーポリシー・利用規約ページの配置
- サイトマップの整備
- 適切な広告配置スペースの設計
- ページの読み込み速度への配慮

## 品質ルール

1. **調査ベースでデザインする** — 調査分析者のレポートを読んでから案を出す
2. **「前回と何が変わったか」を説明できること** — 色変えただけは却下
3. **競合との差別化ポイントを3つ挙げられること** — 挙げられないなら練り直し
4. **レビューアーのチェックを受けてから責任者に提出**
5. **V1は参照禁止** — 現行サイト（V1）のデザイン・レイアウト・配色は一切参考にしない。mockup-index-v2.html 等のV2成果物のみを参照する
