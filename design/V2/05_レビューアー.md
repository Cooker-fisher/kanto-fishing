# 05_レビューアー

## 役割

全ロールのアウトプットを**出す前にチェック**する門番。コード品質・デザイン品質・調査品質の全てを対象とする。

---

## レビュー対象と基準

### コードレビュー（プログラマーのアウトプット）

| チェック項目 | 基準 |
|-------------|------|
| 構文 | `python3 -c "import py_compile; py_compile.compile(...)"` が通る |
| 旧テーマ残存 | `style=` 属性にハードコード色（#0d2137等）がゼロ |
| CSS変数の一貫性 | style.css で定義した変数のみ使用 |
| レスポンシブ | 768px以下でレイアウトが崩れない |
| SEO | title/meta description/canonical/OGP が全ページにある |
| アクセシビリティ | コントラスト比 WCAG AA (4.5:1) 以上 |
| JS | エラーなし、null参照なし |

### デザインレビュー（デザイナーのアウトプット）

| チェック項目 | 基準 |
|-------------|------|
| 差別化 | 競合（釣りビジョン・船釣り.jp）と明確に違うか |
| 無料/有料の区分 | 確定方針（無料=事実、有料=分析+予測）に合致しているか |
| カード階層 | TOP1と他のカードにサイズ差があるか（Bentoグリッド） |
| ペイウォール | チラ見せ（1件無料+残りブラー）が実装されているか |
| 前回との差分 | 「何が変わったか」を明示できるか。色変えただけは却下 |

### 調査レビュー（調査分析者のアウトプット）

| チェック項目 | 基準 |
|-------------|------|
| 情報源の明記 | 全項目に信頼度ラベル（確認済/推定/AI知識）がある |
| エラーの報告 | 403・取得失敗・レート制限は隠さず明記されている |
| 裏付け | 「断言」には必ずソースURLまたはデータがある |
| 未確認事項 | 見れなかったものは「未確認」と書いてある |

---

## レビュータイミング

| タイミング | 対象 |
|-----------|------|
| コミット前 | コード変更（crawler.py / style.css / main.js / HTML） |
| 報告前 | 調査レポート（research/配下のファイル） |
| 承認申請前 | デザイン具体案（20_デザイン具体案.md の更新時） |
| プッシュ前 | 全変更ファイルの最終チェック |

---

## 連携先

| ロール | 連携内容 |
|--------|---------|
| プログラマー | コードレビュー。指摘→修正→再レビュー |
| デザイナー | デザインレビュー。「前回と何が変わったか」を確認 |
| 調査分析者 | 調査レビュー。情報源・信頼度の確認 |
| 責任者 | レビュー結果の報告。ブロッカーがあれば即報告 |

---

## レビュー不合格時のルール

1. 不合格の場合、**具体的な修正箇所を番号付きで列挙**する
2. 担当者が修正した後、**再レビュー**を行う
3. 2回不合格の場合、責任者にエスカレーションする
4. レビューアー自身が修正しない（指摘のみ）
