## コードレビュー基準（crawler.py / style.css / main.js / HTML）

| チェック項目 | 基準 | 確認方法 |
|-------------|------|---------|
| 構文 | `python -m py_compile crawler.py` が通る | Bash実行 |
| V1コード残存 | V1固有の関数名・クラス名が混入していない | Grep |
| 旧テーマ色残存 | `style=` 属性にハードコード色がゼロ | `#0d2137` 等をGrep |
| CSS変数の一貫性 | `var(--xxx)` 形式のみ使用 | ハードコード色をGrep |
| レスポンシブ | 768px以下でレイアウトが崩れない | style.cssをRead |
| SEO | title / meta description / canonical / OGP が全ページにある | 生成HTMLをRead |
| JS | null参照・未定義変数がない | scriptタグをRead |

---

## デザインレビュー基準

| チェック項目 | 基準 |
|-------------|------|
| 差別化 | 競合（釣りビジョン・船釣り.jp）と明確に違うか |
| 無料/有料の区分 | 無料=事実（釣果数値）、有料=分析+予測。混在していないか |
| ペイウォール | 1件目無料 + 残りCSSブラー方式になっているか |
| CSS変数 | `20_デザイン具体案.md` の `:root` 変数を使っているか |
| 前回との差分 | 何が変わったかを説明できるか。色変えだけは却下 |

---

## 調査レビュー基準

| チェック項目 | 基準 |
|-------------|------|
| 情報源の明記 | 全項目に信頼度ラベル（確認済/推定/AI知識）がある |
| エラーの報告 | 403・取得失敗は隠さず明記されている |
| 裏付け | 断言にはソースファイルパスまたはURLがある |
| 未確認事項 | 見れなかったものは「未確認」と書いてある |

---

## レビュータイミング

| タイミング | 対象 |
|-----------|------|
| コミット前 | コード変更（crawler.py / style.css / main.js / HTML） |
| 報告前 | 調査レポート（research/配下） |
| 承認申請前 | デザイン具体案（20_デザイン具体案.md の更新時） |
| プッシュ前 | 全変更ファイルの最終チェック |

## レビュー結果フォーマット

### 合格
```
✅ レビュー合格
対象: <ファイル名>
確認項目: 全X項目クリア
```

### 不合格
```
❌ レビュー不合格
対象: <ファイル名>

1. [項目名] <具体的な問題> — <修正方針>
2. [項目名] <具体的な問題> — <修正方針>

修正後に再レビューを依頼してください。
```

2回不合格の場合は責任者にエスカレーション。
